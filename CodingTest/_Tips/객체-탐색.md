# 낯선 JS 객체 탐색법

> TL;DR: 공식 문서 없이도 `console.dir` → `getOwnPropertyNames` → 블랙박스 테스트 3단계로 낯선 객체의 API를 파악할 수 있다. 코테에서 모르는 메서드를 빠르게 탐색하는 데 유용.

---

## 1. 시각적 탐색 (Visual Inspection)

개발 환경의 UI로 객체의 '설명서'를 직관적으로 들여다본다.

### `console.dir(obj)`

- **핵심**: `console.log()`가 단순 문자열·요약본이라면, `dir`은 객체의 속성과 메서드를 **상세한 트리 구조**로 보여준다.
- **활용**: 콘솔 출력의 `▶` 화살표 → **`[[Prototype]]`** 또는 **`__proto__`** 항목을 펼치면, 그 객체가 상속받아 사용할 수 있는 모든 메서드 리스트가 나옴.

### 인텔리센스 / 자동완성

- 브라우저 콘솔이나 VS Code에서 객체 뒤에 `.`을 찍거나 `Ctrl + Space`를 누르면 호출 가능한 함수 목록이 팝업.
- 함수에 마우스를 올리면 매개변수 힌트도 확인.

---

## 2. 프로그래밍적 탐색 (Programmatic Inspection)

코드 명령으로 객체 특징을 텍스트/배열로 직접 반환. Node.js 같은 터미널 환경에서 유용.

### 속성 이름 전부 추출

```javascript
Object.getOwnPropertyNames(Object.getPrototypeOf(obj));
```

원리: 객체의 부모(원형, Prototype)에 있는 메서드·속성 이름을 **문자열 배열**로 한눈에 추출.

### 타입 / 소속 확인

```javascript
typeof obj;              // "object" (기본 데이터 타입)
obj instanceof Date;     // true/false (특정 클래스의 인스턴스인지)
obj.constructor.name;    // "Date" (생성자 함수 이름)
```

---

## 3. 블랙박스 테스트 (Trial & Error)

내부 구조를 모르는 상태에서 예측되는 메서드에 다양한 값을 넣고 출력을 관찰하는 귀납적 방법.

1. **가설 세우기**: `getOwnPropertyNames`에서 `getFullYear`, `getMonth`, `getDay` 같은 이름을 발견 → "연·월·요일 가져오는 기능"이라고 가정.
2. **입출력 관찰**:

```javascript
let sample = new Date(2016, 0, 1); // 기준 데이터 명시
console.log(sample.getDay());      // 출력값으로 의미 추론 (예: 5 = 금요일?)
```

결과 `5`가 어떤 의미인지 기준 데이터와 비교해 규칙성(0=일, 1=월…)을 찾아낸다.

---

## 요약

> 낯선 자바스크립트 객체를 만나면:
> 1. `console.dir()`로 프로토타입 열어보기
> 2. `Object.getPrototypeOf()` + `getOwnPropertyNames()`로 메서드 이름 추출
> 3. 기준 데이터를 넣고 `console.log()`로 실행 결과 비교

공식 문서 없이도 코드를 파악하는 표준 흐름.
