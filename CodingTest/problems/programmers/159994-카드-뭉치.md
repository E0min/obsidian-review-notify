# [프로그래머스] 카드 뭉치 (159994)

> TL;DR: 두 카드 덱의 앞 카드를 순서대로 꺼내 goal 순서를 완성할 수 있는지 확인. goal 현재 항목이 두 덱 모두의 앞 카드와 다르면 즉시 "No" — 그리디 조기 종료 패턴.

---

## 문제

- `cards1`, `cards2`: 순서가 고정된 두 카드 덱
- `goal`: 완성해야 하는 카드 순서
- 각 덱에서는 **앞에서부터 순서대로**만 꺼낼 수 있음
- goal의 순서대로 카드를 완성할 수 있으면 `"Yes"`, 아니면 `"No"`

```
cards1 = ["i", "drink", "water"]
cards2 = ["want", "to"]
goal   = ["i", "want", "to", "drink", "water"]
→ "Yes"

goal 순서: i(c1✅) → want(c2✅) → to(c2✅) → drink(c1✅) → water(c1✅)
```

---

## 내 풀이 (Two-pointer + Greedy) ✅

```javascript
function solution(cards1, cards2, goal) {
    let c1 = 0;
    let c2 = 0;
    let g  = 0;

    while ((c1 < cards1.length || c2 < cards2.length) && g < goal.length) {
        if (cards1[c1] === goal[g]) {
            c1++;
            g++;
        } else if (cards2[c2] === goal[g]) {
            c2++;
            g++;
        } else {
            return "No"; // 두 덱 모두 앞 카드가 goal과 다름 → 즉시 실패
        }
    }
    return "Yes";
}
```

**시간복잡도**: O(n) — goal 길이만큼 최대 1회 순회

**핵심 아이디어**:
- `cards1[c1]`와 `cards2[c2]` 중 하나가 `goal[g]`를 만족하면 해당 포인터 전진
- 둘 다 안 되면 더 이상 진행 불가 → 즉시 `"No"` 반환 (그리디 조기 종료)
- 모든 goal을 소화하면 `"Yes"`

---

## 핵심 인사이트 — a/~a 출력 구조

이 문제는 **`a / ~a` 구조**: "No"가 되는 조건(두 덱 모두 불일치)을 찾아 즉시 반환. 나머지는 전부 "Yes".

```
if (실패 조건) return "No";   ← 하나라도 실패하면 즉시 종료
...
return "Yes";                 ← 끝까지 살아남으면 성공
```

→ 출력 구조 3가지(a/~a, a/b, 다중) 전체 판별법: **[[../../_Tips/출력조건-구조파악]]** (SSOT)

## 실수 포인트

```javascript
// ❌ == 대신 === 사용 권장 (타입 강제 변환 방지)
if (cards1[c1] == goal[g])   // 동작하지만 습관적으로 ===
if (cards1[c1] === goal[g])  // ✅
```
