# [프로그래머스] 가장 가까운 같은 글자 (142086)

> TL;DR: 현재 문자와 가장 최근에 등장한 동일 문자 사이 거리. Map으로 "마지막 등장 인덱스"를 추적하면 O(n).

---

## 문제

문자열 `s`의 각 위치 `i`에 대해, `i`보다 앞에 `s[i]`와 같은 문자가 있으면 그 중 **가장 가까운 것과의 거리**를 반환. 없으면 `-1`.

- 예: `"banana"` → `[-1, -1, -1, 2, 2, 2]`

---

## 내 최초 풀이 (스택 / O(n²))

```javascript
function solution(s) {
    var answer = [];
    const sStack = [];
    
    for (let j = 0; j < s.length; j++) {
        if (sStack.length == 0) {
            answer.push(-1);
        } else {
            let flag = -1;
            sStack.forEach((e, i) => {
                if (s[j] == e) {
                    flag = j - i;  // 거리 = 현재 인덱스 - 스택 내 인덱스
                }
            });
            answer.push(flag);
        }
        sStack.push(s[j]);
    }    
    return answer;
}
```

**시간복잡도**: O(n²) — 매 위치마다 스택 전체를 순회

**문제점**: `forEach`로 스택 전체를 탐색하면서 마지막으로 찾은 인덱스(`flag`)를 계속 덮어씀.  
→ "가장 가까운" = 마지막 등장이므로 논리는 맞지만 **불필요하게 전체 순회**.

---

## 내 두 번째 풀이 (Object / O(n)) ✅

```javascript
function solution(s) {
    var answer = [];
    let lastSeen = {};
    
    for (let i = 0; i < s.length; i++) {
        let char = s[i];
        if (lastSeen[char] !== undefined) { // 이전에 등장한 적 있다면
            answer.push(i - lastSeen[char]);
        } else { // 처음 등장한다면
            answer.push(-1);
        }
        lastSeen[char] = i; // 현재 인덱스로 갱신
    }
    return answer;
}
```

**시간복잡도**: O(n) — Map이 아닌 Object를 사용했지만 해시 테이블 기반으로 동일하게 O(1) 탐색

**성장 포인트**: 첫 풀이(스택 전체 순회 O(n²))에서 스스로 핵심을 잡아냄.  
→ "이전 등장 여부" = 마지막 인덱스만 기억하면 된다는 아이디어를 체득.

**`!== undefined` 체크**: `lastSeen[char] === 0`인 경우(인덱스 0에 등장)를 falsy로 놓치지 않기 위해 `!== undefined`가 필요. 단순 `if (lastSeen[char])`이면 버그.

---

## 최적 풀이 (Map + 함수형 / O(n))

```javascript
function solution(s) {
    const lastIndex = new Map(); // 문자 → 마지막 등장 인덱스
    
    return [...s].map((char, i) => {
        const result = lastIndex.has(char) ? i - lastIndex.get(char) : -1;
        lastIndex.set(char, i); // 현재 인덱스로 갱신
        return result;
    });
}
```

**시간복잡도**: O(n) — 각 문자를 한 번씩만 처리

**핵심 아이디어**: "가장 가까운 = 마지막 등장"이므로 Map에 최신 인덱스만 유지하면 됨. 스택 전체를 뒤질 필요 없음.

---

## 풀이 성장 비교

| | 1차 (스택) | **2차 (Object)** | 최적 (Map + 함수형) |
|--|---------|---------|---------|
| 자료구조 | 배열(스택) | Object | Map |
| 탐색 방식 | forEach 전체 순회 | 해시 직접 접근 | 해시 직접 접근 |
| 시간복잡도 | O(n²) | **O(n)** | O(n) |
| 코드 스타일 | 명령형 | 명령형 | 함수형 |
| 핵심 아이디어 습득 | ❌ | ✅ | ✅ |

**흡수할 아이디어**: "가장 가까운 이전 등장" 유형 → **Object/Map으로 마지막 인덱스 1개만 추적**이 표준 패턴. 배열에 전체 이력을 쌓는 건 대부분 과설계.

**주의**: 인덱스를 값으로 쓸 때는 `!== undefined` 또는 `in` 연산자로 존재 여부 확인. `if (obj[key])` 는 `0`을 falsy로 처리해 버그 유발.
