# 책상 앞 풀이 순서

> TL;DR: 제약 조건으로 복잡도 역산 → 알고리즘 유형 매핑 → 손으로 예제 추적 → 구현 → 테스트. 막히면 [[공부법#1시간-룰]].

---

## 1. 제약 조건 분석 (N 범위 → 복잡도 역산)

문제 읽는 즉시 N 크기를 보고 허용 알고리즘을 역산한다.

| N 범위 | 허용 복잡도 | 대표 알고리즘 |
|--------|-------------|---------------|
| N ≤ 10 | O(N!) | 완전탐색, 백트래킹 |
| N ≤ 20 | O(2ᴺ) | 비트마스킹, 완전 탐색 |
| N ≤ 100 | O(N³) | Floyd-Warshall |
| N ≤ 1,000 | O(N²) | 이중 루프, 단순 DP |
| N ≤ 100,000 | O(N log N) | 정렬, 이분탐색, 세그트리 |
| N ≤ 1,000,000 | O(N) | 투포인터, 슬라이딩 윈도우, 그리디 |
| N ≤ 10,000,000 | O(log N) | 이분탐색 단독 |

> 시간 제한 1초 ≈ **10⁸ 연산** 기준.

---

## 2. 알고리즘 유형 체크리스트

```
□ 완전탐색 (Brute Force)
  - 경우의 수가 작을 때 (N ≤ 10~12)
  - 백트래킹으로 가지치기

□ 그리디 (Greedy)
  - "매 순간 최선"이 전체 최선이 되는 구조
  - 증명 필수: 교환 논증 or 구간 조건

□ 다이나믹 프로그래밍 (DP)
  - 부분 문제가 중복 + 최적 부분 구조
  - 점화식 먼저, 그 다음 배열 초기화
  - 1D → 2D 순서로 시도

□ BFS / DFS
  - 최단 경로 (가중치 없음) → BFS
  - 연결 요소, 경로 탐색 → DFS
  - 상태 공간 탐색 → BFS (큐)

□ 다익스트라 / 벨만-포드 / Floyd-Warshall
  - 단일 출발점 최단 (음수 X) → 다익스트라
  - 음수 포함, 음수 사이클 감지 → 벨만-포드
  - 모든 쌍 최단 (N ≤ 500) → Floyd-Warshall

□ 투포인터 / 슬라이딩 윈도우
  - 연속 부분 배열 합/조건
  - 정렬된 배열에서 두 수의 합

□ 이분탐색 (Binary Search)
  - 단조 증가/감소 함수에서 값 찾기
  - "최대 최솟값", "최소 최댓값" → 이분탐색 + feasibility check

□ 자료구조
  - 스택: 후입선출, 괄호, 모노톤 스택
  - 큐: 선입선출, BFS
  - 힙: 우선순위 큐, K번째 최솟값
  - 해시: O(1) 탐색, 빈도 카운트
  - 세그먼트 트리: 구간 합·최솟값 갱신 O(log N)

□ 그래프 심화
  - 위상정렬 (DAG, 순서 의존성)
  - 유니온-파인드 (연결 요소, 사이클 탐지)
  - 최소 신장 트리 (Kruskal, Prim)
```

---

## 3. 풀이 단계

```
1. 문제 읽기 + 예제 I/O 직접 손으로 추적 (5분)
2. 제약 조건 파악 → 허용 복잡도 역산 (1분)
3. 유형 체크리스트 대조 → top-2 후보 선택 (2분)
   ↳ 동시에 [[../_Tips/출력조건-구조파악]] — 출력이 a/~a면 여사건 접근
4. 손으로 예제 풀기 → 점화식/알고리즘 검증 (5분)
   ↳ 이미지화 또는 수도코드로 논리 흐름 정리
5. 구현 (엣지 케이스 리스트 옆에 적어두기)
   ↳ JS면 [[../_Tips/js-구현-팁]] 참고
6. 예제 테스트 → 제출
```

핵심: **수도코드 → 실제 코드** 순서. 머릿속에서 바로 코드 짜면 꼬임.

## 4. 구현 시 기억할 것

### 정렬 우선 원칙

> **데이터가 주어지면 웬만하면 정렬하고 시작하는 게 빠르다.**

그리디·스케쥴링·이분탐색 — 대부분 정렬이 전처리 첫 단계.

### DP 작성 패턴

```python
# 1. dp 배열 정의 (의미 먼저)
# dp[i] = i번째 원소까지 고려했을 때 최대 부분합

# 2. 초기값 (base case)
dp[0] = nums[0]

# 3. 점화식
for i in range(1, n):
    dp[i] = max(nums[i], dp[i-1] + nums[i])

# 4. 정답 추출
return max(dp)
```

**DP 메모리 최적화**: 직전 상태만 필요하면 1D 배열 → 변수 2개로 압축.

```python
# O(N) 공간 → O(1) 공간
prev, curr = dp[0], 0
for i in range(1, n):
    curr = max(nums[i], prev + nums[i])
    prev = curr
```

### DP를 쓰는 조건

- **최대 혹은 최소**를 물어볼 때
- 그리디로 정해지지 않고, **다음 상태가 이전 상태에 의존**할 때
- 풀이 중에 **중복 계산**이 존재할 때
- 바텀업 방식: 시작 상태 존재 + 점화식 형태 필수

#### Knapsack 방향성
- **마지막부터 돌기** → 중복 선택 불가 (0/1 Knapsack)
- **처음부터 돌기** → 중복 선택 허용 (Unbounded Knapsack)

### 자주 쓰는 자료구조

- **우선순위 큐** — SJF, 다익스트라, 프림
- **유니온-파인드** — 크루스칼, 사이클 탐지

---

## Pitfalls

- **N 범위 무시**: 제약 안 보고 구현부터 → O(N²) 짜놓고 N=10⁶에서 TLE
- **점화식 선 정의 생략**: 의미 정의 없이 dp[i][j] 짜면 반드시 꼬임
- **정렬 안 하고 투포인터**: 투포인터는 정렬된 배열 전제
- **BFS에서 방문 체크 누락**: 무한 루프 → 중복 방문은 큐에 넣기 전에 체크
- **오버플로우**: N=10⁵, 값=10⁹ → 합이 10¹⁴ → int 범위 초과, long/BigInt 사용
- **입력 크기별 자료형**: Python은 무제한 정수라 자유, Java/C++는 long 명시
