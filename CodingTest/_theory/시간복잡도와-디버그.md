# 🪜 시간복잡도와 디버그

> TL;DR: Big-O 표기법으로 알고리즘 효율 측정 (Worst case 기준). 1초 ≈ 1억 연산. 디버그는 초기화·범위·변수·자료형 4가지 함정 우선.

# 시간 복잡도

시간 복잡도에는 다음과 같은 세 가지 유형이 존재한다.

- **빅 - 오메가** $\Omega(n)$ : Best case의 연산 횟수를 나타낸 표기법
- **빅 - 세타** $\Theta(n)$ : Average case의 연산 횟수를 나타낸 표기법
- **빅 - 오** $O(n)$ : Worst case의 연산 횟수를 나타낸 표기법

코딩 테스트에서는 **빅 - 오** 표기법을 기준으로 수행시간을 계산하는 것이 좋다. 다양한 케이스를 모두 통과해야 하므로 최악일 때를 염두에 두는 시간 복잡도를 선택하여 모든 케이스가 통과하도록 해야 한다.

연산 횟수는 **1초에 1억번 연산**하는 것을 기준으로 생각하며, 연산 횟수 = 알고리즘 시간복잡도 × 데이터의 크기이다.

## 예시
N개의 수가 주어졌을 때 오름차순으로 정렬하는 프로그램을 작성해야 하는 경우, 문제에서 1 ≤ N ≤ 1,000,000 의 범위를 만족하며 제한시간이 2초일 경우:

- 버블 정렬의 Worst case는 $O(n^2)$ 이므로 1,000,000,000,000번의 연산을 하여 1000초가 걸린다.
- 병합 정렬의 Worst case는 $O(n\log_2 n)$ 이므로 1,000,000 × $\log_2(1,000,000)$ = 약 20,000,000번의 연산. 200,000,000 연산 보다 시간이 적게 걸린다.

→ **병합 정렬을 사용해야 한다.**

## 시간 복잡도 계산 규칙
1. 상수는 무시해도 가능하나, 너무 크다면 고려해야함
2. 최고 차항의 차수가 시간 복잡도의 기준이 된다.
