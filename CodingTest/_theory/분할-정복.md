# 🏏 분할 정복

> TL;DR: 문제를 하위 문제로 쪼개 해결 → 합침. problem/solution은 그대로, **size만 바뀜**. 충분히 작으면 adhoc. Master Theorem으로 시간복잡도 계산.

# 분할 정복

- 문제를 하위 문제들로 쪼개서 하위 문제들을 해결하는 방식
- 기존의 문제 해결을 위해 하위 문제들에 대한 해결을 합친다
- **problem과 solution은 바뀌지 않는다. 그러나 문제의 size가 바뀐다.**
- 문제 x와 해결책 y가 있을 때, 문제 x를 쪼개서 x_i로 만들고 각 해결책 y_i를 구한 다음 y_i를 합쳐 전체 솔루션 y를 구한다.

```c
function DivideConquer(x){
    if (x is sufficiently small) then {
        return adhoc(x)  // x가 충분히 작을 때는 분할 정복 말고 적절한 알고리즘 수행
    }
    for ( i=1 to l ) do {
        y_i <- DiveConquer(x_i)
    }
    Combine all the y_i to obtain a solution y
}
```

### adhoc (특별한 목적) algorithm
- 문제를 해결할 수 있는 간단한 알고리즘
- 작은 크기 인스턴스에서는 효율적이지만 큰 사이즈는 고려하지 않음

## 분할 정복의 시간 복잡도 (Master Theorem)

분할 정복 알고리즘의 재귀적 시간 복잡도를 풀기 위한 방법. 다음과 같은 재귀 관계를 가진 알고리즘에 적용:

$$T(n) = aT\left(\frac{n}{b}\right) + f(n)$$

- $n$: 문제의 크기
- $a$: 분할된 부분 문제의 수
- $b$: 각 부분 문제 크기를 줄이는 비율 (부분 문제 크기는 $n/b$)
- $f(n)$: 문제를 분할/병합하는 데 걸리는 작업 시간

### 3가지 Case

1. **Case 1**: $f(n)$이 $af(n/b)$ 보다 **빠르게 증가**하면, $T(n) = \Theta(f(n))$.   조건: $l < b^k$

2. **Case 2**: $f(n)$이 $af(n/b)$와 **같은 속도**로 증가하면 (즉, $f(n) = \Theta(n^{\log_b a} \log^k n)$, $k \geq 0$), $T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$.   조건: $l = b^k$

3. **Case 3**: $f(n)$이 $af(n/b)$보다 **느리게 증가**하고 일정한 $c < 1$에 대해 $af(n/b) \leq cf(n)$이면, $f(n) = o(n^{\log_b a})$.   조건: $l > b^k$

Master Theorem이 적용되지 않는 재귀 관계도 있기 때문에, 그런 경우에는 다른 방법(반복 대입, 재귀 트리 등)을 사용해야 한다.

## DP와의 차이
- **분할 정복**: 하위 문제들이 **독립적** (overlapping subproblems 없음). 예: 병합 정렬
- **DP**: 하위 문제들이 **중복** (overlapping subproblems). 메모이제이션으로 중복 제거. → [[Dynamic-Programming]]
