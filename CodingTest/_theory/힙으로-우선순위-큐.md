# 🪫 힙으로 우선순위 큐 만들기

> TL;DR: 힙은 완전 이진 트리 기반으로 최댓값/최솟값을 O(1)에 접근, 삽입/삭제를 O(log n)에 수행. buildHeap은 O(n). 우선순위 큐의 최적 구현체.

---

# 1. 힙이란?

힙(Heap)은 **이진 트리(Binary Tree)** 기반의 자료구조로, **최댓값 또는 최솟값을 빠르게 찾고 추출**하는 데 최적화된 구조이다. 주로 **우선순위 큐**를 구현할 때 사용되며, 삽입과 삭제 연산의 시간 복잡도가 **O(log n)**으로 효율적이다.

## 힙의 주요 특성

**1. 완전 이진 트리(Complete Binary Tree)**
마지막 레벨을 제외한 모든 레벨이 꽉 차 있으며, 마지막 레벨의 노드는 왼쪽에서부터 채워진 트리.

**2. 힙 특성(Heap Property)**
- **최소 힙(Min-Heap)**: 부모 노드 ≤ 자식 노드. 루트가 최솟값.
- **최대 힙(Max-Heap)**: 부모 노드 ≥ 자식 노드. 루트가 최댓값.

```
// 최소 힙          // 최대 힙
      1                  10
     / \                /  \
    3   6              9    8
   / \ / \            / \  / \
  5  9 8 10          7  6 5   4
```

**3. 배열 표현** — 완전 이진 트리이므로 배열로 직접 표현 가능:
- 부모: `Math.floor((i - 1) / 2)`
- 왼쪽 자식: `2 * i + 1`
- 오른쪽 자식: `2 * i + 2`

## 힙의 이점 요약

| 연산 | 배열/리스트 | 힙 |
|------|------------|-----|
| 최댓값/최솟값 접근 | O(n) | **O(1)** |
| 삽입 | O(1) | O(log n) |
| 삭제(최대/최소) | O(n) | **O(log n)** |
| 균형 유지 | X | **자동** |

# 2. 우선순위 큐

우선순위 큐(Priority Queue)는 **우선순위가 높은 요소**를 먼저 처리하는 자료구조.

| 특징 | 일반 큐(Queue) | 우선순위 큐(Priority Queue) |
|------|--------------|---------------------------|
| 처리 방식 | FIFO | 우선순위가 높은 요소 먼저 |
| 삽입 | O(1) | O(log n) |
| 삭제 | O(1) | O(log n) |
| 최솟값/최댓값 접근 | O(n) | O(1) |

힙을 사용하면 최고 우선순위 요소를 **O(1)**에 접근, **O(log n)**에 추출 가능하므로 우선순위 큐 구현에 최적.

---

# 3. MinHeap 구현 (HeapifyUp + HeapifyDown)

- **HeapifyUp** (Percolate Up): 새 요소 **삽입** 시 끝→위로 부모와 비교 정렬. O(log n)
- **HeapifyDown** (Percolate Down): 루트 **추출** 후 위→아래로 자식과 비교 정렬. O(log n)

```javascript
class PriorityQueue { // MinHeap
    constructor(arr = []) {
        this.heap = arr.slice();
        this.buildHeap(); // O(n)으로 초기 힙 구성
    }

    // buildHeap: 마지막 부모부터 역순으로 heapifyDown → O(n)
    buildHeap() {
        const n = this.heap.length;
        for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
            this.heapifyDown(i);
        }
    }

    insert(value) {
        this.heap.push(value);
        this.heapifyUp(this.heap.length - 1);
    }

    extractMin() {
        if (this.heap.length === 0) return null;
        if (this.heap.length === 1) return this.heap.pop();

        const min = this.heap[0];
        this.heap[0] = this.heap.pop(); // 마지막 요소를 루트로 이동
        this.heapifyDown(0);
        return min;
    }

    heapifyUp(index) {
        let currentIndex = index;
        while (currentIndex > 0) {
            let parentIndex = Math.floor((currentIndex - 1) / 2);
            if (this.heap[currentIndex] < this.heap[parentIndex]) {
                this.swap(currentIndex, parentIndex);
                currentIndex = parentIndex;
            } else {
                break;
            }
        }
    }

    heapifyDown(index) {
        let currentIndex = index;
        const length = this.heap.length;

        while (true) {
            let left = 2 * currentIndex + 1;
            let right = 2 * currentIndex + 2;
            let smallest = currentIndex;

            if (left < length && this.heap[left] < this.heap[smallest]) smallest = left;
            if (right < length && this.heap[right] < this.heap[smallest]) smallest = right;

            if (smallest !== currentIndex) {
                this.swap(currentIndex, smallest);
                currentIndex = smallest;
            } else {
                break;
            }
        }
    }

    swap(i, j) {
        [this.heap[i], this.heap[j]] = [this.heap[j], this.heap[i]];
    }

    size() { return this.heap.length; }
}
```

# 4. MaxHeap 구현

```javascript
class MaxHeap {
    constructor() {
        this.heap = [];
    }

    insert(value) {
        this.heap.push(value);
        this.heapifyUp();
    }

    extractMax() {
        if (this.heap.length === 0) return null;
        if (this.heap.length === 1) return this.heap.pop();

        const max = this.heap[0];
        this.heap[0] = this.heap.pop();
        this.heapifyDown();
        return max;
    }

    heapifyUp() {
        let index = this.heap.length - 1;
        while (index > 0) {
            let parentIndex = Math.floor((index - 1) / 2);
            if (this.heap[parentIndex] < this.heap[index]) {
                this.swap(parentIndex, index);
                index = parentIndex;
            } else {
                break;
            }
        }
    }

    heapifyDown() {
        let index = 0;
        const length = this.heap.length;
        while (true) {
            let left = 2 * index + 1;
            let right = 2 * index + 2;
            let largest = index;

            if (left < length && this.heap[left] > this.heap[largest]) largest = left;
            if (right < length && this.heap[right] > this.heap[largest]) largest = right;

            if (largest !== index) {
                this.swap(index, largest);
                index = largest;
            } else {
                break;
            }
        }
    }

    swap(i, j) {
        [this.heap[i], this.heap[j]] = [this.heap[j], this.heap[i]];
    }
}

const maxHeap = new MaxHeap();
maxHeap.insert(10); maxHeap.insert(5); maxHeap.insert(30); maxHeap.insert(20);
console.log(maxHeap.heap);      // [30, 20, 10, 5]
console.log(maxHeap.extractMax()); // 30
```

# 5. buildHeap이 O(n)인 이유

초기 배열을 힙으로 만드는 두 방법:

| 방법 | 시간복잡도 | 이유 |
|------|-----------|------|
| 요소 하나씩 insert | O(n log n) | 각 insert마다 heapifyUp O(log n) |
| **buildHeap** | **O(n)** | 높이별 작업량 합산 결과 |

**직관**: 완전 이진 트리에서 리프 노드(높이 0)가 전체의 절반. heapifyDown 비용 0. 상위 레벨로 올라갈수록 노드 수가 절반씩 감소하면서 비용 증가. 수학적 합산 결과 O(n).

예: n=7 트리에서 각 레벨별 총 작업량 = 0 + 2 + 8 = 10 → O(n)

예시 `[50, 1, 2, 3, 4, 5, 6]`:
- **buildHeap**: `[1, 3, 2, 50, 4, 5, 6]` — O(n)
- **하나씩 insert**: `[1, 3, 2, 50, 4, 5, 6]` — O(n log n), 동일 결과이지만 더 느림

# 6. 응용 — 스코빌 지수 문제

```javascript
function solution(scoville, K) {
    const pq = createMinHeap(scoville); // buildHeap 사용
    let count = 0;

    while (pq.length >= 1) {
        const s = extractMin(pq);
        if (s >= K) return count;
        if (pq.length === 0) return -1;

        const b = extractMin(pq);
        insert(pq, s + b * 2);
        count++;
    }
    return -1;
}
// [1, 2, 3, 9, 10, 12], K=7 → 2
```
