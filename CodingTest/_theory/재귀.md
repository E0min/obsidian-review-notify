# 📼 재귀

> TL;DR: 자기 자신을 호출. base case + recursive case + 콜 스택. 패턴 7가지(직접/간접/꼬리/비꼬리/다중/트리/서로). return 뒤 호출 vs 그냥 호출 차이는 추가 작업 유무.

## 1. 재귀 알고리즘이란?

**정의**: 하나의 문제를 작은 하위 문제로 분할하여 해결하는 알고리즘 설계 기법으로, 자기 자신을 호출하여 반복적인 문제 해결을 시도. 각 하위 문제는 원래 문제와 동일한 구조를 가지며, **기본 조건(base case)**을 만나면 재귀 호출이 중단되고 결과가 반환된다.

**일반적인 특징**
- **하위 문제로 분할**: 큰 문제를 동일 유형의 작은 하위 문제들로 분할
- **기본 조건**: 재귀 호출을 중단시키는 종료 조건 필수
- **재귀 호출**: 함수가 자기 자신을 호출
- **스택 사용**: 호출 스택으로 현재 상태 저장
- **문제 해결**: 하위 문제 결과를 모아 원래 문제 해결

```java
Algorithm RecursionExample(n)
{
    if (n == 0)
        return 1  // 기본 조건 (Base case)
    else
        return n * RecursionExample(n - 1)  // 재귀 호출
}
```

`RecursionExample(3)` 흐름:
1. `3 * RecursionExample(2)` 호출
2. `RecursionExample(2)` → `2 * RecursionExample(1)`
3. `RecursionExample(1)` → `1 * RecursionExample(0)` → `1`
4. 역순 곱: `3! = 6`

## 2. 재귀호출의 7가지 패턴

### 1. 직접 재귀 (Direct Recursion)
함수가 자기 자신을 직접 호출.

```javascript
function factorial(n) {
    if (n === 0) return 1;
    return n * factorial(n - 1);
}
```

### 2. 간접 재귀 (Indirect Recursion)
함수 A가 함수 B를 호출하고, B가 다시 A를 호출.

```javascript
function A(n) {
    if (n > 0) { console.log("A:", n); B(n - 1); }
}
function B(n) {
    if (n > 0) { console.log("B:", n); A(n - 1); }
}
A(5);
```

### 3. 꼬리 재귀 (Tail Recursion)
함수의 마지막 동작이 자기 자신 호출. 많은 언어에서 최적화 가능 (Tail Call Optimization).

```javascript
function tailRecursion(n, acc = 1) {
    if (n === 0) return acc;
    return tailRecursion(n - 1, n * acc);
}
```

### 4. 비꼬리 재귀 (Non-Tail Recursion)
재귀 호출 뒤에 추가 연산이 있을 때.

```javascript
function nonTailRecursion(n) {
    if (n === 0) return 1;
    return n * nonTailRecursion(n - 1);  // 호출 후 곱 연산
}
```

### 5. 다중 재귀 (Multiple Recursion)
하나의 함수에서 두 번 이상 재귀 호출.

```javascript
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

### 6. 트리 재귀 (Tree Recursion)
다중 재귀의 한 형태. 재귀 호출 트리 생성. 피보나치 수열이 대표.

### 7. 서로 재귀 (Mutual Recursion)
여러 함수가 서로를 재귀적으로 호출. A→B→C→A.

## 3. return을 쓰는 경우 vs 그냥 호출하는 경우

### 그냥 재귀 호출 (재귀 호출 뒤 추가 작업)
```javascript
function countDown(n) {
    if (n <= 0) { console.log("Blastoff!"); return; }
    console.log(n);
    countDown(n - 1);
    console.log("After recursive call:", n);  // 호출 후 작업
}
```
- **재귀 호출 이후에 추가 작업**
- 호출 끝난 뒤 스택 프레임 유지 + 추가 작업 후 종료
- 실행 흐름: 깊이 내려갔다가 돌아오며 "After recursive call: 1" → "2" → "3"

### return 뒤 재귀 호출
```javascript
function factorial(n) {
    if (n === 0) return 1;
    return n * factorial(n - 1);  // 결과 반환
}
```
- 재귀 호출의 **결과 값을 바로 반환**
- 호출이 끝나면 다른 작업 없음
- 꼬리 재귀와 비슷하지만 결과 값 연산이 추가될 수 있음

## 4. 재귀 알고리즘 문제를 풀 때 생각할 것

### 1. 기저 사례 (Base Case)
- 재귀의 종료 조건. 모든 재귀 호출은 반드시 이 기저 사례에 도달
- 잘못 설정되면 **무한 재귀 → 스택 오버플로우**
- 발생 가능한 여러 케이스에 대해 빈틈없이 상황 분리

### 2. 재귀 관계 정의 (Recursion Relation) — 일반식 만들기
- 큰 문제를 어떻게 작은 문제로 나누는지 명확히
- 예: `n-1`을 줄여가거나, 문제 범위를 절반으로 줄여가거나

### 3. 종료 조건까지 가는 경로 확인
- 매 재귀 호출이 점점 더 작은 하위 문제로 수렴하여 기저 사례로 도달하는지
- 무한 루프 방지

### 4. 메모이제이션 (Memoization)
- 동일한 하위 문제 반복 호출 시 결과 저장 → 중복 계산 방지
- 피보나치 등 다중 재귀에서 핵심
- 자세히: [[Dynamic-Programming]]

### 5. 스택 깊이(Stack Depth) 및 메모리
- 너무 깊은 재귀 → 스택 오버플로우 (RangeError)
- 입력 크기 큰 경우 재귀 깊이 제한 설정 또는 반복적 방법 고려
