# 정렬 알고리즘

- 정렬 알고리즘은 **안정적인 알고리즘**과 그렇지 않은 알고리즘으로 나눌 수 있다. 안정적인 정렬은 값이 같은 원소의 순서가 정렬 후에도 같은 것이고 안정적이지 않은 알고리즘은 보장할 수 없다.
- **내부 정렬**: 정렬할 모든 데이터를 하나의 배열에 저장할 수 있는 경우에 사용하는 알고리즘
- **외부 정렬**: 정렬할 데이터가 많아서 하나의 배열에 저장할 수 없는 경우에 사용하는 알고리즘
- 정렬 알고리즘의 핵심은 **교환 / 선택 / 삽입**이다.
- 버블 정렬의 시간 복잡도는 $O(N^2)$이다.

## 버블 정렬

이웃한 두 원소의 대소 관계를 비교하여 필요에 따라 교환을 반복하는 알고리즘. 단순 교환 정렬이라고도 한다.

```python
def Bubble(li: list):
    for a in range(len(li)-1):
        for i in range(0, len(li)-1-a):
            count[0] += 1
            if li[i] > li[i+1]:
                li[i], li[i+1] = li[i+1], li[i]

if __name__ == '__main__':
    sort = [5, 8, 1, 23, 13, 45, 35]
    count = [0]
    Bubble(sort)
    # 21번의 교환 과정을 거침
```

### 개선된 방안 (조기 종료)
- 정렬이 다 된 시점에 중단하면 비교 연산 시간을 줄일 수 있다.
- `for i in range(...)` 실행 시 교환을 하지 않는다면 그 배열은 이미 정렬이 다 됐다고 판단.

```python
def Bubble(li: list):
    for a in range(len(li)-1):
        exchange = 0
        for i in range(0, len(li)-1-a):
            if li[i] > li[i+1]:
                li[i], li[i+1] = li[i+1], li[i]
                exchange += 1
        if exchange == 0:
            break
# 15번의 교환 과정
```

### 셰이커 정렬 (Cocktail Sort)
단방향으로 진행하던 알고리즘을 좌우 번갈아가며 실행.

```python
def shaker(li: list):
    left = 0
    right = len(li)-1
    while left < right:
        exchange = 0
        for i in range(left, right):  # 좌 -> 우
            if li[i] > li[i+1]:
                li[i], li[i+1] = li[i+1], li[i]
                exchange += 1
        right -= 1
        if exchange == 0: break
        for i in range(right, left, -1):  # 우 -> 좌
            if li[i] < li[i-1]:
                li[i], li[i-1] = li[i-1], li[i]
                exchange += 1
        left += 1
        if exchange == 0: break
```

## 단순 선택 정렬

가장 작은 원소부터 선택해 알맞은 위치로 옮기는 작업을 반복.

```python
def simplechoice(li: list):
    start = 0
    end = len(li)
    while start < end:
        index = 0
        min = li[start]
        for i in range(start, end):
            if li[i] <= min:
                min = li[i]
                index = i
        li[start], li[index] = li[index], li[start]
        start += 1
```

## 단순 삽입 정렬

주목한 원소보다 더 앞쪽에서 알맞은 위치로 삽입하며 정렬. 단순 선택과 비슷하지만 값이 가장 작은 원소를 선택하지 않는다. 이미 정렬됐거나 거의 끝나갈 때 매우 빠르다. 삽입 위치가 멀면 이동 횟수가 많아진다.

```python
def simpleInsert(li: list):
    length = len(li)
    for i in range(1, length):
        point = i
        while point > 0 and li[point] < li[point-1]:
            li[point-1], li[point] = li[point], li[point-1]
            point -= 1
```

## 셸 정렬

단순 삽입은 정렬이 거의 끝나갈 때 빠르지만, 삽입 위치가 먼 경우 그 위치까지 비교·교환이 많다. 셸 정렬은 원소를 **그룹별로 나누어 각 그룹별로 정렬한 후 합쳐** 이동 횟수를 줄인다. `gap` 크기는 배수가 안 되게 하는 것이 좋다.

```python
def shell_sort(li: list):
    gap = len(li) // 2
    while gap > 0:
        for i in range(gap, len(li)):
            point = i
            while point > 0 and li[point] < li[point-gap]:
                li[point], li[point-gap] = li[point-gap], li[point]
                point = point - gap
        gap //= 2
```

## 퀵 정렬

기준이 되는 **피벗(pivot)**을 정하고 피벗을 기준으로 그룹을 나누는 것을 반복해 그룹에 수가 하나 남을 때까지 반복.

```python
def quickSort(li: list, left, right):
    pivot = li[(left+right)//2]
    pl = left
    pr = right
    while pl <= pr:
        while pivot > li[pl]: pl += 1   # A
        while pivot < li[pr]: pr -= 1   # B
        if pl <= pr:
            li[pl], li[pr] = li[pr], li[pl]   # C
            pl += 1
            pr -= 1
    if left < pr: quickSort(li, left, pr)
    if right > pl: quickSort(li, pl, right)
```

- A: 피벗 오른쪽으로 가야 하는 원소를 찾기 위해 `pl` 증가
- B: 피벗 왼쪽으로 가야 하는 원소를 찾기 위해 `pr` 감소
- C: 두 원소 교환

### 시간복잡도
- **Best**: pivot이 중간값일 때 $O(n \log n)$
- **Worst**: pivot이 최댓값/최솟값일 때 $O(n^2)$

## 병합 정렬

배열을 앞부분/뒷부분 두 그룹으로 나누어 각각 정렬한 후 병합. **안정 정렬**. 분할 정복 기반.

```python
def mergeSort(li: list, start, end) -> None:
    center = (start+end)//2
    if start < end:
        mergeSort(li, start, center)
        mergeSort(li, center+1, end)
        lc = start
        rc = center + 1
        buf = []
        while lc <= center and rc <= end:
            if li[lc] < li[rc]:
                buf.append(li[lc]); lc += 1
            else:
                buf.append(li[rc]); rc += 1
        while lc <= center: buf.append(li[lc]); lc += 1
        while rc <= end: buf.append(li[rc]); rc += 1
        li[start:end+1] = buf
```

### 시간 복잡도
- 병합: $O(n)$
- 단계: $\log n$
- 전체: $O(n \log n)$

## 힙 정렬

힙은 '**부모의 값이 자식보다 항상 크다**'(또는 작다)는 조건을 만족하는 완전 이진 트리.

힙에서 원소 `a[i]`는 다음 관계를 가진다:
- 부모: `a[(i-1) // 2]`
- 왼쪽 자식: `a[i*2 + 1]`
- 오른쪽 자식: `a[i*2 + 2]`

힙 정렬 과정:
1. 루트(최댓값)를 꺼낸다
2. 마지막 원소를 루트로 이동
3. 루트에서 시작해 자식과 비교하며 자리를 바꾸는 heapify-down

자세한 힙 구현: [[힙으로-우선순위-큐]]

```python
def make_heap(li, parent):
    lc = parent*2+1
    rc = parent*2+2
    if lc < len(li): make_heap(li, lc)
    if rc < len(li): make_heap(li, rc)
    if lc < len(li) and li[parent] < li[lc]:
        li[parent], li[lc] = li[lc], li[parent]
        make_heap(li, lc)
    if rc < len(li) and li[parent] < li[rc]:
        li[parent], li[rc] = li[rc], li[parent]
        make_heap(li, rc)

def heap_sort(li):
    make_heap(li, 0)
    buf = []
    while li:
        buf.append(li[0])
        li[0], li[-1] = li[-1], li[0]
        li.pop()
        make_heap(li, 0)
    return buf
```

## 도수 정렬 (Counting Sort)

원소의 대소 관계를 판단하지 않고 빠르게 정렬. **도수 분포표**와 **누적 도수 분포표**를 사용. 정수만 가능하며, 최소~최대 범위만큼 배열 필요.

### 원리
1. **도수 분포표** 작성 → 누적 도수 분포표 작성
2. 누적 도수 분포표로 위치 파악 (중복 처리 시 -1)

### 시간 복잡도
- 데이터 카운트: $O(N)$
- 도수표 작성: $O(K)$
- 정렬 배열 작성: $O(N)$
- 전체: $O(N + K)$
- 공간: $O(N + K)$

$K$가 $N$에 비해 극단적으로 크면 비효율적이지만, $K$가 작으면 매우 빠른 정렬.

```python
def counting_sort(li):
    maxofli = max(li)
    fredistable = [0]*(maxofli+1)
    Cumulativesum = [0]*(maxofli+1)
    outputli = [0]*len(li)
    for i in li: fredistable[i] += 1
    for i in range(maxofli+1):
        if i == 0: Cumulativesum[i] = fredistable[i]
        else: Cumulativesum[i] = fredistable[i]+Cumulativesum[i-1]
    for i in li:
        outputli[Cumulativesum[i]-1] = i
        Cumulativesum[i] -= 1
    return outputli
```
