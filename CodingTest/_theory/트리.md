# 🥅 트리

> TL;DR: 계층적 구조의 비순환 그래프. 노드 N개면 간선 N-1개. 이진 트리(BST) + 완전 이진 트리(힙)가 핵심. 트리는 그래프의 특수 형태.

---

# 1. 트리(Tree)란?

트리(Tree)는 **계층적 구조**를 표현하는 자료구조로, **노드(Node)**와 **간선(Edge)**으로 이루어져 있다. **루트(Root)** 노드에서 출발하여 부모-자식 관계를 나타낸다. 트리는 **사이클이 없는 그래프**(DAG의 특수 형태)로, 파일 시스템, 조직도, 이진 탐색 트리 등에서 활용된다.

## 트리의 구성 요소

| 용어 | 설명 |
|------|------|
| **루트 노드** | 최상위 노드, 부모 없음 |
| **단말 노드(Leaf)** | 자식 노드가 없는 노드 |
| **내부 노드** | 자식 노드를 가진 노드 |
| **레벨(Level)** | 루트로부터의 거리 (루트 = 0) |
| **깊이(Depth)** | 루트부터 특정 노드까지의 경로 길이 |
| **높이(Height)** | 루트부터 가장 깊은 노드까지의 거리 |

## 트리의 특징
- **사이클 없음** (비순환 그래프)
- **N개의 노드 → N-1개의 간선**
- **두 노드를 연결하는 경로가 유일**

---

# 2. 트리의 종류

### 1. 이진 트리(Binary Tree)
**자식 노드가 최대 2개**인 트리.

### 2. 이진 탐색 트리(Binary Search Tree, BST)
**왼쪽 자식 < 부모 < 오른쪽 자식** 규칙을 가진 이진 트리. 효율적인 탐색과 정렬에 사용.

### 3. 균형 이진 트리(Balanced Binary Tree)
모든 노드의 왼쪽·오른쪽 서브트리 높이 차이가 **1 이하**. 예: AVL 트리, 레드-블랙 트리.

### 4. 완전 이진 트리(Complete Binary Tree)
모든 레벨이 완전히 채워지고, 마지막 레벨은 **왼쪽에서 오른쪽으로** 순차적으로 채워짐. 예: **힙(Heap)**.

### 5. 이진 힙(Binary Heap)
완전 이진 트리 기반:
- **최대 힙(Max Heap)**: 부모 ≥ 자식
- **최소 힙(Min Heap)**: 부모 ≤ 자식
- 우선순위 큐 구현에 사용 → [[힙으로-우선순위-큐]]

# 3. 구현 방법 1 — 이진 트리 (BST)

```javascript
class Node {
    constructor(data) {
        this.data = data;
        this.left = null;
        this.right = null;
    }
}

class BinaryTree {
    constructor() {
        this.root = null;
    }

    insert(data) {
        const newNode = new Node(data);
        if (!this.root) {
            this.root = newNode;
            return;
        }

        let current = this.root;
        while (true) {
            if (data < current.data) {
                if (!current.left) { current.left = newNode; break; }
                current = current.left;
            } else {
                if (!current.right) { current.right = newNode; break; }
                current = current.right;
            }
        }
    }

    // 중위 순회(In-order): 왼쪽 → 부모 → 오른쪽 (오름차순 출력)
    inOrder(node = this.root) {
        if (node) {
            this.inOrder(node.left);
            console.log(node.data);
            this.inOrder(node.right);
        }
    }
}

const tree = new BinaryTree();
tree.insert(10); tree.insert(5); tree.insert(15); tree.insert(7);
tree.inOrder(); // 5, 7, 10, 15
```

**시간 및 공간 복잡도:**
- 삽입/탐색: 평균 **O(log N)**, 최악(편향 트리) O(N)
- 공간: **O(N)**

# 4. 구현 방법 2 — 힙 (MaxHeap)

```javascript
class MaxHeap {
    constructor() {
        this.heap = [];
    }

    insert(value) {
        this.heap.push(value);
        this.bubbleUp();
    }

    bubbleUp() {
        let index = this.heap.length - 1;
        const element = this.heap[index];

        while (index > 0) {
            let parentIndex = Math.floor((index - 1) / 2);
            let parent = this.heap[parentIndex];
            if (element <= parent) break;
            this.heap[index] = parent;
            index = parentIndex;
        }
        this.heap[index] = element;
    }

    extractMax() {
        const max = this.heap[0];
        const end = this.heap.pop();

        if (this.heap.length > 0) {
            this.heap[0] = end;
            this.sinkDown(0);
        }
        return max;
    }

    sinkDown(index) {
        const length = this.heap.length;
        const element = this.heap[index];

        while (true) {
            let left = 2 * index + 1;
            let right = 2 * index + 2;
            let swap = null;

            if (left < length && this.heap[left] > element) swap = left;
            if (right < length && this.heap[right] > (swap ? this.heap[left] : element)) {
                swap = right;
            }
            if (!swap) break;

            this.heap[index] = this.heap[swap];
            index = swap;
        }
        this.heap[index] = element;
    }
}

const heap = new MaxHeap();
heap.insert(10); heap.insert(20); heap.insert(15);
console.log(heap.extractMax()); // 20
```

**삽입/삭제: O(log N)**, **공간: O(N)**

# 5. 트리 순회 방법

| 순회 방법 | 순서 | 특징 |
|----------|------|------|
| 전위 순회(Pre-order) | 부모 → 왼쪽 → 오른쪽 | 트리 복사, 직렬화 |
| **중위 순회(In-order)** | 왼쪽 → 부모 → 오른쪽 | BST에서 오름차순 정렬 |
| 후위 순회(Post-order) | 왼쪽 → 오른쪽 → 부모 | 트리 삭제, 공간 계산 |
| 레벨 순회(BFS) | 레벨별 | 너비 우선, 큐 사용 |

---

# 6. 트리 vs 그래프

| 특징 | 트리(Tree) | 그래프(Graph) |
|------|-----------|-------------|
| **구조** | 계층적, 사이클 없음 | 사이클 있을 수 있음 |
| **간선 수** | N개 노드 → N-1개 간선 | 제한 없음 |
| **루트 노드** | 존재 | 일반적으로 없음 |
| **방향성** | 부모→자식 방향 있음 | 방향/무방향 모두 가능 |
| **순환성** | 불가능 | 가능 |
| **관계** | **그래프의 특수 형태** | 더 일반적 |

트리는 **계층 구조** 표현에 유리, 그래프는 **복잡한 연결** 표현에 사용.
