# 🎀 그래프

> TL;DR: 노드(정점) + 간선(엣지)으로 이루어진 자료구조. 인접 리스트(희소 그래프)와 인접 행렬(밀집 그래프)로 구현. 다익스트라/벨만-포드/플로이드-워셜/크루스칼/프림의 기반.

---

# 1. 그래프의 구성 요소

1. **노드(Node/Vertex)**: 그래프의 각 개체. A, B, C 또는 1, 2, 3 등.
2. **간선(Edge)**: 노드 간의 관계/연결. 방향이 있을 수도, 없을 수도 있음.
3. **방향성**: 무방향(A-B = B-A) vs 방향(A→B ≠ B→A)
4. **가중치(Weight)**: 간선의 비용 또는 거리. 예: A --(5)-- B

---

# 2. 그래프의 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| 무방향 그래프 | 간선에 방향 없음. A-B면 B-A도 가능 | 친구 관계 |
| 방향 그래프 | 간선에 방향. A→B는 A에서 B만 가능 | 팔로우 관계 |
| 가중치 그래프 | 각 간선에 비용/거리 부여 | 도로 지도 |

---

# 3. 구현 방법 1 — 인접 리스트 (Adjacency List)

각 노드를 키로, 연결된 노드들을 배열로 저장. **희소 그래프에 적합**.

```javascript
class Graph {
    constructor() {
        this.adjacencyList = {};
    }

    addNode(node) {
        if (!this.adjacencyList[node]) {
            this.adjacencyList[node] = [];
        }
    }

    // 무방향 그래프
    addEdge(node1, node2) {
        this.adjacencyList[node1].push(node2);
        this.adjacencyList[node2].push(node1);
    }

    // 방향 그래프
    addDirectedEdge(node1, node2) {
        this.adjacencyList[node1].push(node2);
    }

    printGraph() {
        for (let node in this.adjacencyList) {
            console.log(node + " -> " + this.adjacencyList[node].join(", "));
        }
    }
}

const graph = new Graph();
graph.addNode('A'); graph.addNode('B'); graph.addNode('C');
graph.addEdge('A', 'B');
graph.addEdge('A', 'C');
graph.addDirectedEdge('B', 'C');
graph.printGraph();
// A -> B, C
// B -> A, C
// C -> A
```

**시간 및 공간 복잡도:**
- 공간: **O(N + E)** — 연결된 노드만 저장
- 간선 확인: **O(N)** — 리스트 순회 필요

---

# 4. 구현 방법 2 — 인접 행렬 (Adjacency Matrix)

2차원 배열로 노드 간 연결 상태 표현. **밀집 그래프에 적합**.

```javascript
class GraphMatrix {
    constructor(size) {
        this.matrix = Array.from({ length: size }, () => Array(size).fill(0));
        this.size = size;
    }

    // 무방향 그래프
    addEdge(node1, node2) {
        this.matrix[node1][node2] = 1;
        this.matrix[node2][node1] = 1;
    }

    // 방향 그래프
    addDirectedEdge(node1, node2) {
        this.matrix[node1][node2] = 1;
    }

    printGraph() {
        console.log(this.matrix);
    }
}

const graphMatrix = new GraphMatrix(3);
graphMatrix.addEdge(0, 1);
graphMatrix.addDirectedEdge(1, 2);
graphMatrix.printGraph();
// [[0,1,0],[1,0,1],[0,0,0]]
```

**시간 및 공간 복잡도:**
- 공간: **O(N²)** — 모든 정점 쌍을 저장
- 간선 확인: **O(1)** — 인덱스 직접 접근

---

# 5. 인접 리스트 vs 인접 행렬 비교

| 특징 | 인접 행렬 | 인접 리스트 |
|------|----------|------------|
| **메모리** | O(N²) — 크다 | O(N + E) — 효율적 |
| **간선 확인** | **O(1)** — 빠름 | O(N) — 느림 |
| **간선 추가/제거** | O(1) | O(1) |
| **희소 그래프** | 메모리 낭비 큼 | **적합** |
| **밀집 그래프** | **적합** | 비효율적 |
| **구현 난이도** | 간단 | 상대적으로 복잡 |

# 6. 알고리즘별 자료구조 추천

| 알고리즘 | 추천 자료구조 | 이유 |
|---------|-------------|------|
| [[다익스트라]] | **인접 리스트** | 연결된 정점만 탐색, 희소 그래프에 효율적 |
| [[벨만-포드]] | **인접 리스트** | 모든 간선 확인, 간선 중심 탐색 |
| [[플로이드-워셜]] | **인접 행렬** | 모든 정점 쌍 계산, O(V³), 직관적 |
| [[Kruscal]] | **인접 리스트** | 간선 중심 알고리즘 |
| [[Prim]] | **인접 리스트** | 인접 정점 선택 방식, 희소 그래프 효율 |

**결론**:
- **희소 그래프** → 인접 리스트 (다익스트라, 벨만-포드, 크루스칼, 프림)
- **밀집 그래프** → 인접 행렬 (플로이드-워셜)
