# 🎚️ 그리디

> TL;DR: '현재 상황에서 지금 당장 좋은 것만 고르는 방법'. 부분 최적 ≠ 항상 전체 최적. Solution()·Feasible()·Selection() 3 함수. 코테에서 정렬과 짝으로 자주 출제.

## 노션 원본
- [그리디 (Notion)](https://www.notion.so/3dac7a40d83247548c2ad9be1d04640b)

---

- 그리디 알고리즘은 단순하지만 가장 강력한 문제 해결 방법으로 **'현재 상황에서 지금 당장 좋은 것만 고르는 방법'**이다.
- 다른 알고리즘과 다르게 미리 외우고 있지 않아도 풀 수 있는 문제가 많다. 다익스트라 같은 특이한 알고리즘이 있지만, 암기로 그리디의 모든 유형을 해결할 수 없다.
- 코딩테스트의 그리디 문제는 **현재 상황에서 가장 좋아 보이는 것만 선택**해서 풀 수 있는 **창의성**을 요구한다.
- '가장 큰 순서대로', '가장 작은 순서대로'와 같이 문제에서 가장 좋아 보이는 것에 대한 **기준을 알게 모르게 제시**한다. 그래서 **정렬 알고리즘과 짝**을 이뤄 자주 출제된다.

## Optimization Problem (최적화 문제)

- 최적화(주로 최소화 또는 최대화)는 조건을 만족시키는 함수
- **실행 가능한 해결책 (Feasible solution)**: 조건을 만족시키는 모든 해결 방법
- **최적의 해결책 (Optimal solution)**: 최적화 상황을 만족하는 해결 방법

## 그리디 알고리즘

- **현재 주어진 정보**를 기반으로 **미래의 결과에 대해 걱정하지 않고** 결정
- 주로 **최적화 문제**를 해결하는 데 사용

### 일반적인 특징
- 현재 선택 가능한 최적의 방법으로 문제 해결. **그러나 부분 최적이 항상 전체 최적은 아님**
- 후보 집합 C
- 알고리즘 진행 중 두 집합 누적:
  - **SET1**: 이미 고려되어 선택된 후보들
  - **SET2**: 고려되었지만 거부된 후보들

```java
Algorithm Greedy(C)  // C는 candidates의 집합
{
    S <- ∅  // S는 solution 집합 (초기 공집합)
    While(C ≠ ∅ AND not Solution(S))
    {
        x <- Selection(C)  // C에서 유망한 애를 선택
        C <- C - {x}       // x는 C에서 제거
        if (Feasible(S ∪ {x})) then
        {
            S <- S U {x}    // Solution 집합에 x 추가
        }
    }
}
```

- `Solutions()`: 주어진 후보 집합이 문제의 해결책을 제공하는지 확인 (최적성은 일시 무시)
- `Feasible()`: 다른 후보를 추가하여 해결책을 얻을 수 있는지 확인
- `Selection()`: 남아 있는 후보 중 어느 것이 가장 **유망**한지
