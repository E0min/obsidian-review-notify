# ⛓️ 플로이드-워셜 알고리즘

> TL;DR: **모든 정점 쌍 간의 최단 경로 (APSP)**. 음수 가중치 가능. 중간 노드 k를 경유해 i→j를 최적화하는 DP. O(V³). 인접 행렬로 구현. 밀집 그래프에 적합.

---

# 1. 정의

플로이드-워셜(Floyd-Warshall)은 **모든 정점 쌍 간의 최단 경로(All-Pairs Shortest Paths, APSP)**를 찾는 알고리즘이다. **동적 프로그래밍** 기반으로 동작하며, **음수 가중치**가 포함된 경우에도 정상 동작한다.

## 특징

| 항목 | 내용 |
|------|------|
| **대상** | 모든 정점 쌍 (APSP) |
| **음수 가중치** | ✅ 가능 |
| **음수 사이클 탐지** | ✅ (대각선 음수면 탐지) |
| **알고리즘 패턴** | **동적 프로그래밍 (DP)** |
| **시간 복잡도** | **O(V³)** |
| **공간 복잡도** | O(V²) |
| **구현 방식** | 인접 행렬 |
| **적합한 그래프** | **밀집 그래프 (Dense)** |

---

# 2. 핵심 아이디어

**점화식**:
```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

- `dist[i][j]`: 정점 i에서 j까지의 최단 경로
- `k`: 중간 경유 정점
- 모든 k에 대해 "k를 경유하는 경로"가 직접 경로보다 짧은지 확인하며 갱신

**최적성 원칙 (Principle of Optimality)**:
i에서 j로 가는 최단 경로 위의 중간 노드 k가 있다면, i→k 경로와 k→j 경로도 각각 최단 경로이다.

---

# 3. 알고리즘 단계

1. **초기화**: `dist[i][j]` = 간선 가중치 (없으면 `Infinity`, 자기 자신은 0)
2. **k = 1 to V**: 각 정점을 중간 경유 노드로 설정
   - **i = 1 to V**, **j = 1 to V**: 모든 정점 쌍에 대해
     - `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])` 갱신

---

# 4. 의사코드

```plaintext
function FloydWarshall(graph):
    dist = graph  // 인접 행렬

    for k from 1 to V:       // 경유 노드
        for i from 1 to V:   // 출발 정점
            for j from 1 to V:   // 도착 정점
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    return dist
```

# 5. 자바스크립트 구현

```javascript
function floydWarshall(graph) {
    const n = graph.length;
    // dist는 graph 복사본 (원본 수정 방지)
    const dist = Array.from({ length: n }, (_, i) => graph[i].slice());

    for (let k = 0; k < n; k++) {         // 경유 노드
        for (let i = 0; i < n; i++) {     // 출발 정점
            for (let j = 0; j < n; j++) { // 도착 정점
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                }
            }
        }
    }
    return dist;
}

// 예제: 0=자기 자신, Infinity=경로 없음
const graph = [
    [0, 3, Infinity, 5],
    [2, 0, Infinity, 4],
    [Infinity, 1, 0, Infinity],
    [Infinity, Infinity, 2, 0]
];

const result = floydWarshall(graph);
console.log(result);
// 모든 정점 쌍 최단 거리 행렬
```

# 6. 최단 경로 알고리즘 3종 비교

| 알고리즘 | 대상 | 음수 가중치 | 음수 사이클 | 복잡도 | 패턴 |
|---------|------|----------|-----------|--------|------|
| [[다익스트라]] | 단일 출발점 | ❌ | ❌ | O(E log V) | 그리디 |
| [[벨만-포드]] | 단일 출발점 | ✅ | ✅ 탐지 | O(VE) | DP |
| **플로이드-워셜** | **모든 쌍** | ✅ | ✅ 탐지 | **O(V³)** | **DP** |

**선택 기준**:
- 출발점이 하나, 음수 없음 → **다익스트라**
- 출발점이 하나, 음수 있음 → **벨만-포드**
- 모든 쌍 최단 경로 필요 → **플로이드-워셜**
