# 🎯 스케쥴링

> TL;DR: 작업 순서 최적화. **시스템 체류 시간 최소화**는 SJF(서비스 시간 오름차순). **마감기한 있는 단위 시간 작업**은 이익 내림차순 + 마감 전 삽입 가능 여부 체크 그리디.

---

# 1. 정의

스케쥴링(Scheduling) 알고리즘은 주어진 작업들을 **최적 순서**로 처리해 목적 함수(대기 시간, 이익, 마감기한 준수)를 최적화하는 문제다.

코딩 테스트에서 주로 등장하는 두 가지 유형:
1. **시스템 체류 시간 최소화** — SJF (Shortest Job First)
2. **마감기한 있는 단위 시간 작업** — 이익 그리디 + 실행 가능성 체크

---

# 2. 유형 1 — 시스템 체류 시간 최소화

## 핵심 원칙

> 서비스 시간이 짧은 작업부터 처리하면 평균 체류 시간이 최소화된다.

서비스 시간 t₁ ≤ t₂ ≤ ... ≤ tₙ 순서로 처리 시:

```
총 체류 시간 = n·t₁ + (n-1)·t₂ + ... + 1·tₙ
```

**이 순서(오름차순)가 유일한 최적해.**

→ 자세한 구현: [[Shortest-Job-First]]

---

# 3. 유형 2 — 마감기한 있는 단위 시간 작업 (Scheduling with Deadlines)

## 문제 정의

- n개 작업, 각 작업은 **단위 시간(1)** 소요
- 각 작업 i: **이익 gᵢ > 0**, **마감기한 dᵢ**
- 조건: 마감기한 내에 처리해야 이익 획득
- 목표: **총 이익 최대화**

## 예시

| 작업 i | 1 | 2 | 3 | 4 | 5 | 6 |
|---------|---|---|---|---|---|---|
| 이익 gᵢ | 20 | 15 | 10 | 7 | 5 | 3 |
| 마감 dᵢ | 3 | 1 | 1 | 3 | 1 | 3 |

이익 내림차순 정렬 → 순서대로 마감 전 삽입 가능 여부 체크:
- 작업 1 (g=20, d=3): 슬롯 3 사용 ✅
- 작업 2 (g=15, d=1): 슬롯 1 사용 ✅
- 작업 3 (g=10, d=1): 슬롯 1 이미 사용, d≤1 슬롯 없음 ❌
- 작업 4 (g=7, d=3): 슬롯 2 사용 ✅
- 결과: [2, 4, 1] 순서, 총 이익 42

# 4. 알고리즘 — Greedy with Deadline Check

```javascript
// 그리디 알고리즘: 이익 내림차순 → 마감 전 빈 슬롯 있으면 삽입
function scheduleWithDeadlines(jobs) {
    // jobs: [[이익, 마감기한], ...]
    // 이익 내림차순 정렬
    jobs.sort((a, b) => b[0] - a[0]);

    const n = jobs.length;
    const maxDeadline = Math.max(...jobs.map(j => j[1]));
    const slots = new Array(maxDeadline + 1).fill(-1); // slots[time]: 그 시간대에 배정된 작업

    let totalGain = 0;
    const scheduled = [];

    for (const [gain, deadline] of jobs) {
        // 마감기한 이전 가장 늦은 빈 슬롯 탐색 (greedy: 가능한 늦게 배정)
        for (let t = deadline; t >= 1; t--) {
            if (slots[t] === -1) {
                slots[t] = gain;
                totalGain += gain;
                scheduled.push([gain, deadline, t]);
                break;
            }
        }
    }

    return { totalGain, scheduled };
}

// 사용 예시
const jobs = [[20, 3], [15, 1], [10, 1], [7, 3], [5, 1], [3, 3]];
console.log(scheduleWithDeadlines(jobs));
// → { totalGain: 42, scheduled: [[20,3,3],[15,1,1],[7,3,2]] }
```

# 5. 고급: Union-Find 기반 O(n log n) 최적화

단순 배열 탐색은 O(n²). Union-Find로 "다음 빈 슬롯"을 효율적으로 찾으면 O(n log n).

```javascript
class SlotFinder {
    constructor(maxDeadline) {
        this.parent = Array.from({ length: maxDeadline + 2 }, (_, i) => i);
    }

    find(x) {
        if (this.parent[x] === x) return x;
        return this.parent[x] = this.find(this.parent[x]);
    }

    // 슬롯 x를 사용하면 다음 빈 슬롯을 x-1로 연결
    use(x) {
        this.parent[x] = this.find(x - 1);
    }
}

function scheduleOptimized(jobs) {
    jobs.sort((a, b) => b[0] - a[0]);
    const maxDeadline = Math.max(...jobs.map(j => j[1]));
    const sf = new SlotFinder(maxDeadline);
    let totalGain = 0;

    for (const [gain, deadline] of jobs) {
        const slot = sf.find(deadline); // 마감기한 이전 가장 늦은 빈 슬롯
        if (slot > 0) {
            totalGain += gain;
            sf.use(slot);
        }
    }

    return totalGain;
}
```

# 6. 복잡도 정리

| 알고리즘 | 시간 복잡도 | 설명 |
|---------|-----------|------|
| SJF (배열 정렬) | O(n log n) | 단순 정렬로 해결 |
| Deadline Scheduling (단순) | O(n²) | 매 작업마다 빈 슬롯 선형 탐색 |
| **Deadline Scheduling (Union-Find)** | **O(n log n)** | Union-Find로 다음 빈 슬롯 추적 |
