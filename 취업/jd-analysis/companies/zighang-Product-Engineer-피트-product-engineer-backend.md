## 본문 (zighang content)

### 소개

피트(Feat)는 문서를 더 잘 보이게 만드는 회사가 아닙니다. 문서가 열리고, 이해되고, 반응이 남는 방식 자체를 바꾸는 회사입니다.

피트페이퍼(Featpaper)는 PDF·PPT·디자인 문서를 링크 기반의 인터랙티브 문서로 전환합니다. 문서를 전달하는 순간부터 열람, 이해, 반응, 다음 액션까지 하나의 흐름으로 연결하는 B2B SaaS 프로덕트입니다.

피트의 백엔드는 단순히 요청을 처리하는 서버가 아닙니다. 제품의 신뢰도와 확장성을 책임지는 비즈니스의 기반 구조입니다. 전략적으로 설계된 제품 경험이 실제 서비스 품질로 이어지려면, 안정적이면서도 확장 가능한 백엔드 구조가 필수적입니다.

[이 포지션이 필요한 이유]
피트페이퍼는 글로벌 사용자를 대상으로 빠르게 성장하고 있습니다. 문서 열람 데이터, 사용자 반응, 외부 연동 등 백엔드 관점에서 복잡도가 계속 증가하는 단계입니다.

이제는 단순히 기능을 추가하는 수준을 넘어섰습니다. 서비스 구조가 앞으로의 확장을 견딜 수 있는지, 비즈니스 실험을 안정적으로 뒷받침할 수 있는지를 본격적으로 점검하고 고도화해야 하는 시점입니다.

Backend Engineer는 전략 팀과 프로덕트 팀이 설정한 방향을 기술적으로 무리 없이 구현하고, 운영과 확장 과정에서도 흔들리지 않도록 만드는 역할입니다.



### 주요업무

• 피트페이퍼 핵심 서비스의 백엔드 아키텍처 설계 및 개선
• 문서 열람, 사용자 행동, 트래킹 데이터 처리 로직 설계
• 안정적인 API 설계 및 운영
• 글로벌 사용자를 고려한 성능, 확장성, 안정성 개선
• 외부 서비스 및 파트너 API 연동 설계 및 관리
• 데이터 저장 구조 설계 및 운영 (로그, 이벤트, 메타데이터 등)
• PM, 프론트엔드, 디자이너와 협업하여 제품 요구사항을 기술 구조로 구현
• 운영 중 발생하는 이슈 분석 및 재발 방지 구조 설계



### 자격요건

• 백엔드 개발 실무 경험 5년 이상, 또는 이에 준하는 역량을 보유하신 분
• 서비스 단위의 백엔드 시스템을 설계·운영해본 경험
• API, 데이터베이스, 인증/권한 구조에 대한 이해
• 트래킹 증가와 데이터 확장을 고려한 설계 경험
• 코드 품질과 운영 안정성에 책임감을 가지는 분
• 기술 선택의 이유를 설명하고, 팀과 논의할 수 있는 분



### 우대사항

• B2B SaaS 또는 데이터 기반 서비스 개발 경험
• 글로벌 서비스 또는 다국어·다지역 환경 운영 경험
• 대용량 트래픽 또는 이벤트 기반 시스템 경험
• 클라우드 인프라(AWS, GCP 등) 환경에서의 운영 경험
• 로그·모니터링·알림 시스템 구축 경험



### 혜택 및 복지

• 글로벌 사용자를 가진 B2B SaaS의 핵심 인프라를 직접 설계하고 개선할 기회
• 제품과 비즈니스 확장에 맞춰 기술 구조를 함께 진화시킬 수 있는 환경
• 단순 구현자가 아닌, 서비스 구조를 설계하는 엔지니어로 성장할 수 있는 기회
