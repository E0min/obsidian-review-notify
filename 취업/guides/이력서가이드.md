# 이력서 작성 가이드 (Resume Guideline)

## 1. 기본적인 프로젝트 작성 형태

**기본 템플릿:**
```markdown
서비스 개요: [한 줄 요약]
팀원: [인원 수]
기술 스택: [사용 기술 나열]

[핵심 문제 해결 성과 1 (줄글 형식)]
[핵심 문제 해결 성과 2 (줄글 형식)]
[핵심 문제 해결 성과 3 (줄글 형식)]

[단순 구현/기본기 증명 (줄글 형식)]
[인프라/백엔드 관련 경험 (줄글 형식)]
```

### 1-1. 핵심 내용 작성 공식 (Narrative Problem Solving)
**"화살표(→)나 괄호 태그(도메인 등)를 사용하지 않고, 자연스러운 줄글로 작성합니다."**
- **작성 원칙**: **[도메인 + 문제 + 해결 + 결과]**의 논리적 흐름은 유지하되, 담백한 문장으로 서술합니다.
- **Bad Examples**:
    - *교육 게시물 리스트 조회(도메인) → 느린 응답 발생(문제) → EHCache 적용(해결) → 평균 응답 속도 50% 개선(결과)*
- **Good Examples**:
    - *교육 게시물 리스트 조회 시 발생하는 응답 지연 문제를 해결하기 위해 EHCache를 적용하여 평균 응답 속도를 50% 개선했습니다.*
    - *대용량 이미지 로딩으로 인한 렌더링 저하를 방지하고자 Intersection Observer 기반의 지연 로딩을 도입하여 초기 로딩 속도를 1.5초 단축했습니다.*
- **분량**: 각 프로젝트당 이러한 핵심 문장이 **3줄 내외** 포함되어야 합니다.

### 1-2. 단순 구현 및 기타 사항
- **작성 공식**: **[기술적 근거]**와 **[구현 내용]**, 그리고 **[기대 효과(Why)]**를 자연스럽게 연결합니다.
- **예시**:
    - *사용자 경험 증대를 위해 OAuth 2.0 기반의 소셜 로그인을 구현하여 회원가입 전환율을 높였습니다.*
    - *Infra 팀과의 원활한 협업을 위해 Supabase의 RLS 정책을 학습하고 기본적인 보안 규칙을 직접 설정했습니다.*

## 2. 요약
- 전체 프로젝트 덩어리는 **2~3개**면 충분합니다.
- **이모티콘**, **화살표(→)**, **과도한 괄호 태그** 사용을 금지합니다.
- 모든 문장은 **"~했습니다 / ~함"** 등의 일관된 어미로 끝맺습니다.
- 핵심은 **Technical Basis(기술적 근거)**와 **Impact(결과)**가 문장 속에 자연스럽게 녹아드는 것입니다.
