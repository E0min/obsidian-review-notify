# 검색 엔진 최적화 (App Router)

> TL;DR: App Router에서는 `metadata` 객체나 `generateMetadata` 함수로 메타태그를 설정하고, JSON-LD로 구조화 데이터를 추가한다.

## What

Next.js App Router는 파일에서 `metadata` 객체 또는 `generateMetadata` 함수를 export하면 자동으로 `<head>` 태그를 생성한다. Page Router의 `next/head` 방식을 대체한다.

## Why it matters

검색 엔진 크롤러는 HTML의 메타태그와 구조화 데이터를 읽어 페이지 순위를 결정한다. SSR/SSG로 렌더링된 HTML에 정확한 메타데이터가 있어야 검색 노출과 소셜 미리보기가 제대로 작동한다.

## How

### 정적 메타데이터

```typescript
// app/page.tsx 또는 app/layout.tsx
import type { Metadata } from 'next';

export const metadata: Metadata = {
  title: '페이지 제목',
  description: '페이지 설명',
  openGraph: {
    title: 'OG 제목',
    description: 'OG 설명',
    images: [{ url: '/og-image.jpg' }],
  },
  twitter: {
    card: 'summary_large_image',
    title: '트위터 제목',
  },
};
```

### 동적 메타데이터 (generateMetadata)

```typescript
// app/product/[id]/page.tsx
import type { Metadata, ResolvingMetadata } from 'next';

type Props = {
  params: { id: string };
};

export async function generateMetadata(
  { params }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  const product = await fetchProduct(params.id);

  return {
    title: product.name,
    description: product.description,
    openGraph: {
      images: [{ url: product.imageUrl }],
    },
  };
}

export default function ProductPage({ params }: Props) {
  return <div>{/* 페이지 내용 */}</div>;
}
```

### 제목 템플릿 (title template)

```typescript
// app/layout.tsx
export const metadata: Metadata = {
  title: {
    template: '%s | My Site',  // 하위 페이지 제목 뒤에 자동 추가
    default: 'My Site',        // title이 없는 페이지의 기본값
  },
};

// app/about/page.tsx
export const metadata: Metadata = {
  title: 'About', // 결과: "About | My Site"
};
```

### 구조화 데이터 (JSON-LD)

Rich Snippet(별점, 가격 등)을 검색 결과에 표시하려면 JSON-LD를 추가한다.

```typescript
// app/product/[id]/page.tsx
export default async function ProductPage({ params }) {
  const product = await fetchProduct(params.id);

  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "Product",
    name: product.name,
    description: product.description,
    offers: {
      "@type": "Offer",
      price: product.price,
      priceCurrency: "KRW",
    },
  };

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      <div>{/* 페이지 내용 */}</div>
    </>
  );
}
```

### robots.txt 및 sitemap 자동 생성

```typescript
// app/sitemap.ts
import { MetadataRoute } from 'next';

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    { url: 'https://example.com', lastModified: new Date() },
    { url: 'https://example.com/about', lastModified: new Date() },
  ];
}

// app/robots.ts
export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: '*', allow: '/' },
    sitemap: 'https://example.com/sitemap.xml',
  };
}
```

## Pitfalls

- `generateMetadata`는 서버 컴포넌트 파일에서만 사용 가능하다. `"use client"` 파일에는 쓸 수 없다.
- 레이아웃과 페이지 모두에 `metadata`가 있으면 페이지의 값이 우선(deep merge) 적용된다.
- `<img>` 태그의 `alt` 속성 누락은 접근성과 SEO 모두에 영향을 미친다. `next/image` 사용 시 `alt`는 필수다.
- 동적 OG 이미지는 `app/opengraph-image.tsx` 파일로 생성할 수 있다 (`ImageResponse` 활용).

## SEO 체크리스트

1. 메타 태그 및 OG 설정 (`title`, `description`, `openGraph`)
2. 동적 페이지별 `generateMetadata` 적용
3. JSON-LD 구조화 데이터 추가
4. SSR / SSG 활용 (크롤러가 JavaScript 실행 없이 내용을 읽을 수 있어야 함)
5. `next/image` 사용 + `alt` 속성 필수 작성
6. 접근성 개선 (`h1`–`h6` 계층 구조, `aria-*` 속성)
7. `sitemap.ts` 및 `robots.ts` 생성
