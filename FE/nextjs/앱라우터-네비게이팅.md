# App Router 네비게이팅

> TL;DR: App Router는 초기 요청에 HTML + RSC 페이로드 + JS 번들을 보내고, 이후 페이지 이동 시엔 RSC 페이로드만 요청해 JS 번들 중복 다운로드를 방지하며, Link의 프리페칭으로 이동 전 데이터를 미리 준비한다.

## What

App Router에서 페이지 이동(네비게이션)이 동작하는 방식과, `Link` 컴포넌트 및 `useRouter`를 활용한 클라이언트 사이드 네비게이션.

## Why it matters

Page Router와 달리 App Router는 **RSC 페이로드**라는 새로운 전송 형식을 도입했다. 이로 인해 페이지 이동 비용이 줄고, 동적 페이지에서는 JS 번들을 지연 로드할 수 있다.

`next/router`(Page Router)와 `next/navigation`(App Router)은 API가 다르므로 혼용하면 오류가 발생한다.

## How

### 초기 요청 vs 페이지 이동 시 비교

| 시점 | 전송 내용 |
|------|---------|
| 초기 접속 | HTML(SSR) + RSC 페이로드 + JS 번들 |
| 페이지 이동 | RSC 페이로드 + 필요 시 추가 JS 번들 |

초기 이후 이동에서는 이미 다운로드된 JS 번들을 재사용하고, RSC 페이로드(서버에서 렌더링된 컴포넌트 트리 데이터)만 요청한다.

### Link 컴포넌트 (선언적 네비게이션)

```javascript
import Link from 'next/link';

export default function Nav() {
  return (
    <nav>
      <Link href="/about">About</Link>
      <Link href="/blog" prefetch={false}>Blog</Link>  {/* 프리페칭 비활성화 */}
    </nav>
  );
}
```

### Link 프리페칭 동작

뷰포트에 Link가 보일 때 자동으로 해당 경로 데이터를 미리 가져온다.

| 페이지 타입 | 프리페칭 내용 |
|------------|------------|
| 정적 페이지 | RSC 페이로드 + JS 번들 모두 프리페칭 |
| 동적 페이지 | RSC 페이로드만 프리페칭 (JS 번들은 이동 시 로드) |

```javascript
// 프리페칭 명시적 제어
<Link href="/about" prefetch={true}>항상 프리페칭</Link>
<Link href="/about" prefetch={false}>프리페칭 비활성화</Link>
// prefetch 미지정 시: 기본값(뷰포트 진입 시 자동)
```

### 프로그래매틱 네비게이션 (useRouter)

이벤트 핸들러 내에서 코드로 페이지를 이동할 때 사용. `next/navigation`에서 import해야 하며, **클라이언트 컴포넌트에서만 사용 가능**하다.

```javascript
'use client';
import { useRouter } from 'next/navigation'; // App Router 전용

export default function LoginButton() {
  const router = useRouter();

  const handleLogin = async () => {
    await loginAPI();
    router.push('/dashboard');     // 이동 (히스토리에 추가)
    // router.replace('/dashboard'); // 이동 (현재 히스토리 대체)
    // router.back();               // 뒤로 가기
    // router.refresh();            // 현재 페이지 서버 데이터 새로고침
  };

  return <button onClick={handleLogin}>로그인</button>;
}
```

### Page Router와 App Router import 비교

```javascript
// ❌ Page Router 방식 — App Router에서 사용하면 오류
import { useRouter } from 'next/router';

// ✅ App Router 방식
import { useRouter, usePathname, useSearchParams } from 'next/navigation';
```

### 유용한 App Router 훅 모음

```javascript
'use client';
import { usePathname, useSearchParams, useParams } from 'next/navigation';

function NavigationInfo() {
  const pathname = usePathname();       // 현재 경로 문자열 (예: '/blog/1')
  const searchParams = useSearchParams(); // URL 쿼리 파라미터
  const params = useParams();           // 동적 경로 파라미터 (예: { id: '1' })

  return (
    <div>
      <p>현재 경로: {pathname}</p>
      <p>검색어: {searchParams.get('q')}</p>
    </div>
  );
}
```

## Pitfalls

- `useRouter`, `usePathname`, `useSearchParams` 등 navigation 훅은 모두 **클라이언트 컴포넌트 전용** — 서버 컴포넌트에서 사용하면 에러
- `next/router`(Page Router)를 App Router 프로젝트에서 import하면 `NextRouter was not mounted` 에러 발생
- `router.refresh()`는 브라우저 새로고침이 아닌 **서버 컴포넌트 데이터만 다시 fetch**하는 동작 — 클라이언트 상태(useState 값 등)는 유지됨
- `useSearchParams()`는 Suspense 경계 안에서 사용해야 함 (Next.js 14+에서 경고 발생)
