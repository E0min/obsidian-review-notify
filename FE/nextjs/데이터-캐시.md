# 데이터 캐시

> TL;DR: Next.js는 서버에서 fetch한 데이터를 자동으로 캐싱하며, `cache: 'no-store'`(항상 최신)과 `next.revalidate`(주기적 갱신) 옵션으로 캐싱 전략을 페치 단위로 세밀하게 제어한다.

## What

Next.js가 서버 컴포넌트의 `fetch` 호출 결과를 서버에 저장하는 메커니즘. 동일한 URL을 반복 요청할 때 외부 API를 다시 호출하지 않고 캐시된 응답을 반환한다.

3가지 캐싱 옵션이 있으며, 각 `fetch` 호출마다 독립적으로 설정할 수 있다.

## Why it matters

서버 컴포넌트는 매 요청마다 실행되므로, 캐싱 없이 외부 API를 매번 호출하면 응답 속도가 느려지고 API 요금이 증가한다. 반대로 모든 데이터를 캐싱하면 실시간성이 필요한 데이터가 갱신되지 않는다.

데이터 특성에 맞는 캐싱 전략을 fetch 단위로 적용해 성능과 최신성을 동시에 확보한다.

## How

### 옵션 1: 기본 캐싱 (정적 데이터)

```javascript
// cache 옵션 미지정 시 자동 캐싱
fetch('https://api.example.com/news')
// 동일 URL은 빌드 이후 또는 캐시 만료 전까지 재요청 안 함
```

변경이 거의 없는 데이터(블로그 글, 공지사항, 뉴스)에 적합.

### 옵션 2: `cache: 'no-store'` (실시간 데이터)

```javascript
// 매 요청마다 항상 새로운 데이터를 가져옴
fetch('https://api.example.com/stock-price', { cache: 'no-store' })
```

캐싱을 완전히 비활성화한다. 주식 가격, 실시간 재고, 사용자별 맞춤 데이터 등에 적합.

### 옵션 3: `next.revalidate` (주기적 갱신, ISR)

```javascript
// 10초마다 백그라운드에서 데이터를 다시 가져옴
fetch('https://api.example.com/products', { next: { revalidate: 10 } })
```

ISR(Incremental Static Regeneration)처럼 동작한다. 캐시된 데이터를 즉시 반환하되, 지정한 시간이 지나면 백그라운드에서 새 데이터를 fetch해 캐시를 갱신한다. 상품 목록, 환율, 날씨 데이터 등에 적합.

### 사용 사례 정리

| 데이터 유형 | 캐싱 옵션 | 예시 |
|------------|---------|------|
| 정적 콘텐츠 | 기본 캐싱 (옵션 없음) | 블로그 글, 회사 소개 |
| 주기적 갱신 | `next: { revalidate: N }` | 상품 목록, 날씨, 환율 |
| 실시간 데이터 | `cache: 'no-store'` | 주식 가격, 사용자 알림 |

### 페이지/레이아웃 단위 캐싱 제어

개별 fetch 대신 페이지 전체의 캐싱 동작을 설정할 수 있다.

```javascript
// app/dashboard/page.js
export const dynamic = 'force-dynamic'; // 페이지 전체를 no-store처럼 동작
// export const revalidate = 60;        // 페이지 전체를 60초마다 갱신
```

### 캐시 무효화 (온디맨드 revalidation)

특정 이벤트(글 수정, 결제 완료 등) 발생 시 캐시를 즉시 무효화할 수 있다.

```javascript
// app/api/revalidate/route.js
import { revalidatePath, revalidateTag } from 'next/cache';

export async function POST(request) {
  revalidatePath('/blog');        // 특정 경로의 캐시 무효화
  // revalidateTag('blog-posts'); // 태그로 묶인 fetch들의 캐시 무효화
  return Response.json({ revalidated: true });
}
```

```javascript
// 태그와 함께 fetch
fetch('https://api.example.com/posts', {
  next: { tags: ['blog-posts'] } // 태그 지정
})
```

## Pitfalls

- `fetch`가 아닌 다른 방법(axios, DB 쿼리 등)으로 가져온 데이터는 Next.js 데이터 캐시가 적용되지 않음 → `unstable_cache`를 사용해야 캐싱 가능
- `cache: 'no-store'`와 `next: { revalidate: 0 }`은 동일하게 동작함
- 개발 모드(`next dev`)에서는 데이터 캐시가 비활성화된 상태로 동작 — 운영 빌드(`next build && next start`)에서 확인해야 실제 캐싱 동작을 볼 수 있음
- 같은 라우트 세그먼트 내에서 `force-dynamic`과 `revalidate` 옵션을 혼용하면 `force-dynamic`이 우선 적용됨
