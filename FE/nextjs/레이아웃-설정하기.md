# App Router 레이아웃 설정하기

> TL;DR: `layout.js`를 디렉토리마다 배치해 중첩 레이아웃을 구성하고, `(groupname)` 라우트 그룹으로 URL을 오염시키지 않고 특정 경로에만 레이아웃을 선택 적용한다.

## What

`layout.js` 파일은 **해당 디렉토리와 모든 하위 경로**에 공통 UI를 감싸는 역할을 한다. 여러 레이아웃을 계층적으로 중첩할 수 있어, URL 구조와 UI 구조를 일치시킬 수 있다.

## Why it matters

Page Router의 `_app.tsx`는 전체 앱에 단 하나의 레이아웃만 허용했다. 특정 섹션(예: 대시보드)에만 사이드바를 추가하려면 각 페이지 컴포넌트 안에서 직접 처리해야 했다.

App Router는 디렉토리마다 `layout.js`를 두어 **구조 자체가 레이아웃 로직을 담게** 한다. 결과적으로 페이지 컴포넌트는 레이아웃을 모르고, 레이아웃은 페이지를 `children`으로만 받는 명확한 역할 분리가 이루어진다.

## How

### 루트 레이아웃 (필수)

```javascript
// app/layout.js — 전체 앱의 루트 레이아웃 (반드시 존재해야 함)
export default function RootLayout({ children }) {
  return (
    <html lang="ko">
      <body>
        <header>공통 헤더</header>
        {children}
        <footer>공통 푸터</footer>
      </body>
    </html>
  );
}
```

### 중첩 레이아웃

```
app/
├── layout.js          → 전체 공통 (헤더, 푸터)
├── page.js            → /
└── dashboard/
    ├── layout.js      → 대시보드 전용 (사이드바)
    ├── page.js        → /dashboard
    └── settings/
        └── page.js    → /dashboard/settings
```

```javascript
// app/dashboard/layout.js
export default function DashboardLayout({ children }) {
  return (
    <div style={{ display: 'flex' }}>
      <nav>사이드바</nav>
      <main>{children}</main>
    </div>
  );
}
```

`/dashboard/settings` 접속 시 렌더링 순서:
`RootLayout` → `DashboardLayout` → `SettingsPage`

### 라우트 그룹 (Route Group)

`(groupname)` 형식의 폴더명은 **URL 경로에 포함되지 않는다.** 특정 경로에만 레이아웃을 적용하거나 관련 파일을 묶을 때 사용한다.

```
app/
├── layout.js             → 루트 레이아웃
├── (marketing)/
│   ├── layout.js         → 마케팅 전용 레이아웃
│   ├── about/page.js     → /about  (URL에 marketing 없음)
│   └── blog/page.js      → /blog
└── (dashboard)/
    ├── layout.js         → 대시보드 전용 레이아웃 (로그인 가드 포함)
    ├── home/page.js      → /home
    └── settings/page.js  → /settings
```

```javascript
// app/(dashboard)/layout.js — 로그인이 필요한 경로에만 적용
export default function DashboardLayout({ children }) {
  return (
    <div>
      <nav>대시보드 네비게이션</nav>
      {children}
    </div>
  );
}
```

### 동작 순서 요약

```
RootLayout (app/layout.js)
  └─ GroupLayout (app/(group)/layout.js)  ← 라우트 그룹이 있을 경우
       └─ NestedLayout (app/section/layout.js)
            └─ Page (app/section/page.js)
```

## Pitfalls

- `app/layout.js`(루트 레이아웃)는 **반드시 존재해야 하며** `<html>`, `<body>` 태그를 포함해야 함
- 레이아웃은 리렌더링되지 않음 — 탐색 시에도 레이아웃 상태가 유지됨 (이는 의도된 동작)
- 라우트 그룹 내에서 동일 URL 경로가 생기면 빌드 에러 발생 (예: `(a)/about`와 `(b)/about` 동시 존재 불가)
- `layout.js`는 해당 세그먼트에서 `page.js`보다 항상 먼저 렌더링됨 — 레이아웃 내 데이터 페칭은 모든 하위 페이지 렌더 전에 실행됨에 주의
