# 라우팅 설정하기 (동적 경로)

> TL;DR: 파일명에 `[id]`나 `[...id]`를 사용하면 URL 파라미터를 동적으로 받을 수 있고, `router.query`로 값을 추출한다.

## What

Next.js Page Router에서 URL 경로의 일부를 변수로 처리하는 **동적 라우팅** 방식이다. 파일명의 대괄호(`[]`) 표기가 핵심이다.

## Why it matters

블로그 포스트(`/post/1`), 상품 상세(`/product/abc`), 유저 프로필(`/user/hong`) 같이 경로의 일부가 데이터에 따라 달라지는 페이지를 파일 하나로 처리할 수 있다.

## How

### 단일 동적 경로 — `[id].tsx`

URL 하나의 세그먼트를 파라미터로 받는다.

```
pages/post/[id].tsx
→ /post/1      (id: "1")
→ /post/abc    (id: "abc")
```

```typescript
// pages/post/[id].tsx
import { useRouter } from 'next/router';

export default function PostPage() {
  const router = useRouter();
  const { id } = router.query; // URL의 id 파라미터 추출

  return <div>포스트 ID: {id}</div>;
}
```

---

### 캐치홀 세그먼트 — `[...id].tsx`

여러 경로 세그먼트를 **배열**로 받는다.

```
pages/post/[...id].tsx
→ /post/1/2/3    (id: ["1", "2", "3"])
→ /post/a/b      (id: ["a", "b"])
```

```typescript
// pages/post/[...id].tsx
import { useRouter } from 'next/router';

export default function PostPage() {
  const router = useRouter();
  const { id } = router.query; // 배열로 반환: ["1", "2", "3"]

  return <div>경로 세그먼트: {Array.isArray(id) ? id.join(' / ') : id}</div>;
}
```

---

### 선택적 동적 경로

`id`가 없는 경우도 처리해야 할 때 조건부 분기를 사용한다.

```typescript
const { id } = router.query;

if (!id) {
  return <div>전체 목록</div>;
}
return <div>ID: {id}</div>;
```

---

### 404 페이지 커스터마이징

`pages/404.tsx` 파일을 생성하면 자동으로 적용된다.

```typescript
// pages/404.tsx
export default function NotFound() {
  return (
    <div>
      <h1>404 - 페이지를 찾을 수 없습니다</h1>
    </div>
  );
}
```

### 동적 라우트 우선순위

```
pages/post/create.tsx     → /post/create   (정적 경로가 우선)
pages/post/[id].tsx       → /post/:id
pages/post/[...id].tsx    → /post/*
```

## Pitfalls

- `router.query`는 **초기 렌더링 시 빈 객체**일 수 있음 → `router.isReady`를 확인하거나 `useEffect`에서 처리
- 캐치홀 세그먼트는 항상 배열로 반환되므로 타입 체크 필요 (`Array.isArray(id)`)
- 같은 경로 세그먼트에 정적 파일과 동적 파일이 겹치면 정적 파일이 우선됨
- `getStaticPaths`와 함께 사용할 때는 `fallback` 옵션 설정 필수
