# Next.js 스타일링

> TL;DR: Next.js는 글로벌 CSS, CSS 모듈, CSS-in-JS 세 가지 방식으로 스타일을 적용하며, 각 방식마다 적용 범위와 제약이 다르다.

## What

Next.js에서 스타일을 적용하는 방법은 크게 세 가지다.

1. **글로벌 CSS**: 앱 전체에 적용되는 전역 스타일
2. **CSS 모듈**: 컴포넌트 단위로 스코프가 격리된 스타일
3. **CSS-in-JS**: JavaScript 코드 안에서 스타일을 작성하는 방식

## Why it matters

스타일링 방식의 선택은 클래스명 충돌, 번들 사이즈, 유지보수성에 직접 영향을 준다. Next.js는 글로벌 CSS import 위치를 `_app.js`로 제한하여 예측 가능한 스타일 로딩 순서를 보장한다.

## How

### 글로벌 CSS

반드시 `pages/_app.js`(또는 `_app.tsx`)에서만 import해야 한다. 다른 컴포넌트 파일에서 import하면 빌드 오류가 발생한다.

```javascript
// pages/_app.js
import '../styles/global.css';

export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />;
}
```

### CSS 모듈

`*.module.css` 파일을 생성하고 컴포넌트에서 import해 `styles.클래스명` 형태로 사용한다. Next.js가 빌드 시 고유한 해시를 붙여 클래스명 충돌을 방지한다.

```javascript
// components/Button.js
import styles from '../styles/Button.module.css';

export default function Button() {
  return <button className={styles.button}>Click me</button>;
}
```

```css
/* styles/Button.module.css */
.button {
  background-color: #5e6ad2;
  color: white;
  padding: 8px 16px;
}
```

### CSS-in-JS

`styled-components` 또는 `@emotion/styled` 같은 라이브러리를 사용한다. SSR 환경에서는 서버 사이드 스타일 주입을 위한 추가 설정이 필요하다.

```javascript
// styled-components 예시
import styled from 'styled-components';

const Button = styled.button`
  background-color: #5e6ad2;
  color: white;
  padding: 8px 16px;
`;
```

## Pitfalls

- 글로벌 CSS를 `_app.js` 외의 파일에서 import하면 **빌드 오류** 발생
- CSS 모듈 클래스명은 JavaScript 식별자 규칙을 따라야 하므로, 하이픈이 포함된 클래스명은 `styles['class-name']` 대괄호 표기법으로 접근
- `styled-components`를 SSR에서 사용할 때 `_document.js`에 서버 사이드 렌더링 설정을 추가하지 않으면 스타일이 깜빡이는 현상(FOUC) 발생
