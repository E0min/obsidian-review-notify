# 인터셉트 라우트

> TL;DR: `(.)` `(..)` `(...)` 규칙으로 기존 경로의 URL을 유지하면서 다른 UI(예: 모달)를 보여준다.

## What

인터셉트 라우트(Intercepting Routes)는 현재 레이아웃 컨텍스트를 유지하면서 특정 경로 요청을 가로채 다른 UI로 렌더링하는 App Router 기능이다. URL은 실제로 변경되지만, 화면은 모달이나 오버레이처럼 표시된다.

### 인터셉트 규칙

| 규칙 | 설명 |
|---|---|
| `(.)` | 현재 레벨의 경로를 가로챈다 |
| `(..)` | 한 단계 위 부모 레벨의 경로를 가로챈다 |
| `(..)(..)` | 두 단계 위 경로를 가로챈다 |
| `(...)` | 루트(`app/`)부터의 경로를 가로챈다 |

## Why it matters

Instagram의 피드에서 사진을 클릭하면 모달로 보이지만, 직접 URL로 접근하면 전체 페이지로 보이는 패턴이 대표적 사용 사례다. 이를 통해 다음 이점을 얻을 수 있다.

- URL이 바뀌므로 공유 가능한 링크 생성
- 기존 레이아웃(피드)을 유지한 채 콘텐츠 표시
- 브라우저 뒤로 가기로 모달 닫기 지원

## How

### 파일 구조 예시 (피드 + 사진 모달)

```
app/
├── layout.js
├── feed/
│   └── page.js              ← 피드 페이지
├── @modal/
│   ├── default.js           ← 모달 없을 때 null 반환
│   └── (.)photo/[id]/
│       └── page.js          ← 피드에서 클릭 시 모달로 렌더링
└── photo/
    └── [id]/
        └── page.js          ← URL 직접 접근 시 전체 페이지
```

### 모달 페이지 구현

```typescript
// app/@modal/(.)photo/[id]/page.tsx
export default function PhotoModal({ params }: { params: { id: string } }) {
  return (
    <div className="modal-overlay">
      <div className="modal-content">
        <h1>Photo ID: {params.id}</h1>
        {/* 사진 상세 내용 */}
      </div>
    </div>
  );
}
```

### 모달 닫기 (router.back)

```typescript
// app/@modal/(.)photo/[id]/page.tsx
"use client";
import { useRouter } from "next/navigation";

export default function PhotoModal({ params }: { params: { id: string } }) {
  const router = useRouter();

  return (
    <dialog open onClick={() => router.back()}>
      <div onClick={(e) => e.stopPropagation()}>
        <h1>Photo ID: {params.id}</h1>
        <button onClick={() => router.back()}>닫기</button>
      </div>
    </dialog>
  );
}
```

### default.js (슬롯 비활성 상태)

```javascript
// app/@modal/default.js
export default function Default() {
  return null; // 모달이 없을 때 빈 상태 유지
}
```

## Pitfalls

- 인터셉트 규칙 `(.)`, `(..)`, `(...)`은 파일시스템 경로 기준이 아니라 라우트 세그먼트 기준이다. 라우트 그룹 `(폴더명)`은 세그먼트로 치지 않는다.
- 병렬 라우트(`@modal`)와 함께 사용해야 레이아웃을 유지하면서 모달을 렌더링할 수 있다. 단독으로는 기존 레이아웃이 대체된다.
- 직접 URL 접근(새 탭, 새로고침)은 인터셉트가 발동하지 않고 원본 경로의 페이지가 렌더링된다.
- `default.js`가 없으면 모달이 닫혔을 때 슬롯에 매칭 경로가 없어 오류가 발생한다.
