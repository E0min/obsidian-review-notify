# 이미지 최적화 (next/image)

> TL;DR: `next/image`의 `<Image>` 컴포넌트는 자동 크기 최적화, Lazy Loading, WebP/AVIF 변환을 기본 제공한다.

## What

`next/image`는 HTML `<img>` 태그를 대체하는 Next.js 내장 컴포넌트다. 성능에 영향을 주는 이미지 관련 작업을 자동으로 처리한다.

### 주요 기능 요약

| 기능 | 설명 |
|---|---|
| 자동 최적화 | 디바이스·네트워크에 맞는 크기로 리사이즈 |
| 포맷 변환 | WebP / AVIF 자동 변환 (브라우저 지원 여부 감지) |
| Lazy Loading | 기본 적용. 뷰포트에 들어올 때 로드 |
| 반응형 | `fill` + `sizes` 속성으로 유동 크기 대응 |
| 우선 로딩 | `priority` 속성으로 LCP 이미지 사전 로드 |
| 블러 효과 | `placeholder="blur"`로 저화질 선행 표시 |
| CDN 캐싱 | 자동 캐싱 및 CDN 활용 |

## Why it matters

이미지는 웹 성능에서 가장 큰 비중을 차지하는 리소스다. `<img>` 태그를 그대로 사용하면 불필요하게 큰 이미지를 내려받거나, LCP(Largest Contentful Paint)가 느려진다. `next/image`는 이를 자동으로 해결해 Core Web Vitals 점수를 높인다.

## How

### 기본 사용

```typescript
import Image from 'next/image';

// 로컬 이미지: width, height 자동 감지
import exampleImg from '/public/example.jpg';
<Image src={exampleImg} alt="예시 이미지" />

// 경로 문자열: width, height 명시 필요
<Image src="/example.jpg" alt="이미지" width={800} height={600} />
```

### 반응형 (fill 속성)

부모 컨테이너 크기에 맞게 채운다. 부모에 `position: relative`가 있어야 한다.

```typescript
<div style={{ position: "relative", width: "100%", height: "400px" }}>
  <Image
    src="/example.jpg"
    alt="반응형 이미지"
    fill
    sizes="(max-width: 768px) 100vw, 800px"
    style={{ objectFit: "cover" }}
  />
</div>
```

### Hero 이미지 우선 로딩 (priority)

LCP 대상 이미지에는 `priority`를 추가해 사전 로드(preload)한다.

```typescript
<Image
  src="/hero.jpg"
  alt="히어로 이미지"
  width={1200}
  height={600}
  priority
/>
```

### 블러 플레이스홀더

로드 전 저화질 블러 이미지를 먼저 보여준다.

```typescript
// 로컬 이미지: blurDataURL 자동 생성
import heroImg from '/public/hero.jpg';
<Image src={heroImg} alt="히어로" placeholder="blur" />

// 외부 이미지: blurDataURL 직접 제공 필요
<Image
  src="https://example.com/photo.jpg"
  alt="외부 이미지"
  width={800}
  height={600}
  placeholder="blur"
  blurDataURL="data:image/jpeg;base64,..."
/>
```

### 외부 이미지 허용 설정

```javascript
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "example.com",
        pathname: "/images/**",
      },
    ],
  },
};

module.exports = nextConfig;
```

## Pitfalls

- `fill` 속성 사용 시 반드시 부모에 `position: relative`와 명시적 크기가 있어야 한다.
- 외부 이미지는 `next.config.js`의 `remotePatterns`에 도메인을 등록하지 않으면 에러가 발생한다.
- `priority` 없이 뷰포트 상단에 큰 이미지를 쓰면 LCP 점수가 낮아진다. 히어로 이미지에는 반드시 `priority`를 붙인다.
- `width`, `height`를 생략하고 `fill`도 없으면 빌드 에러가 발생한다. 둘 중 하나는 필수다.
