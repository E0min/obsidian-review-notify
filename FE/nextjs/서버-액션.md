# 서버 액션
> TL;DR: `'use server'` 디렉티브로 선언한 함수로, 클라이언트에서 호출해도 서버에서 실행되어 API Route 없이 서버 작업을 처리한다.

## What
서버 액션은 클라이언트 컴포넌트에서 일반 함수처럼 호출하지만 실제 실행은 서버에서 이루어지는 함수다. Next.js가 내부적으로 HTTP 엔드포인트를 자동 생성해 클라이언트-서버 간 통신을 처리한다. 개발자는 fetch나 API Route를 직접 작성하지 않아도 된다.

- `'use server'` 디렉티브 필수
- Next.js 13+ App Router 전용
- 파일 최상단 또는 함수 내부에 선언 가능

## Why it matters
- **API Route 불필요**: 별도 `route.ts` 파일을 만들지 않고 컴포넌트 가까이에서 서버 로직을 선언한다.
- **보안 강화**: 서버에서만 실행되므로 DB 쿼리, 시크릿 키 사용 등을 클라이언트에 노출하지 않는다.
- **RSC와 통합**: 서버 액션 실행 후 `revalidatePath` 등으로 서버 컴포넌트 데이터를 즉시 갱신할 수 있다.

## How

### 기본 선언

```javascript
// app/actions.ts
'use server'; // 파일 최상단 선언 → 파일 내 모든 export가 서버 액션

export async function myServerAction(name: string) {
  // DB 접근, 외부 API 호출 등 서버 작업 가능
  return `Hello, ${name}!`;
}
```

### 클라이언트에서 호출

```javascript
// app/components/MyButton.tsx
'use client';
import { myServerAction } from '../actions';

export default function MyButton() {
  async function handleClick() {
    const result = await myServerAction('Next.js');
    console.log(result); // "Hello, Next.js!"
  }

  return <button onClick={handleClick}>클릭</button>;
}
```

### form action과 함께 사용

```typescript
// app/actions.ts
'use server';
import { revalidatePath } from 'next/cache';

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string;
  // DB에 저장
  await db.post.create({ data: { title } });
  // 페이지 캐시 무효화
  revalidatePath('/posts');
}
```

```typescript
// app/new-post/page.tsx
import { createPost } from '../actions';

export default function NewPostPage() {
  return (
    <form action={createPost}>
      <input name="title" placeholder="제목" />
      <button type="submit">등록</button>
    </form>
  );
}
```

## Pitfalls
- **브라우저 API 사용 불가**: `window`, `document`, `localStorage` 등은 서버에서 실행되는 서버 액션 안에서 접근할 수 없다.
- **직렬화 가능한 인자만 전달 가능**: 클라이언트에서 서버 액션으로 넘기는 인자는 JSON으로 직렬화할 수 있어야 한다. 함수나 DOM 요소는 전달 불가.
- **API Routes와의 비교**: 서버 액션은 컴포넌트와 밀접한 단순 작업(폼 제출, 데이터 변경)에 적합하고, API Routes는 외부 서비스에서 호출하거나 복잡한 미들웨어가 필요한 경우에 적합하다.
- **에러 처리**: 서버 액션 내부에서 throw된 에러는 클라이언트에서 try-catch로 잡아야 한다. `error.tsx`는 렌더링 에러용이라 서버 액션 에러는 직접 처리해야 한다.
