# 에러 핸들링 (error.tsx)
> TL;DR: 라우트 세그먼트에서 발생하는 런타임 에러를 포착해 대체 UI를 표시하는 파일 컨벤션으로, React Error Boundary를 Next.js가 자동 구성한다.

## What
특정 라우트 디렉토리에 `error.tsx`를 만들면 해당 경로(및 하위)에서 렌더링 중 발생하는 에러를 catch하고 사용자에게 에러 UI를 보여준다. try-catch를 모든 서버 컴포넌트에 직접 작성하는 대신, 파일 하나로 에러 경계를 선언적으로 설정한다.

- 반드시 `'use client'` 클라이언트 컴포넌트
- `error` prop: 발생한 Error 객체
- `reset` prop: 에러 복구 시도 함수

## Why it matters
- 에러가 발생한 컴포넌트만 대체 UI로 교체되고 나머지 페이지는 정상 작동한다 (Error Boundary 특성).
- 사용자에게 흰 화면 대신 명확한 에러 메시지와 재시도 버튼을 제공할 수 있다.
- 페이지마다 일일이 try-catch를 쓰지 않아도 된다.

## How

```typescript
// app/dashboard/error.tsx
'use client';

export default function Error({
  error,
  reset,
}: {
  error: Error;
  reset: () => void;
}) {
  return (
    <div>
      <h1>오류 발생!</h1>
      <p>{error.message}</p>
      <button onClick={() => reset()}>재시도</button>
    </div>
  );
}
```

### 에러 복구 3가지 방법

**1. `reset()` — 클라이언트 컴포넌트만 재렌더링**

```typescript
// Error Boundary 내부의 클라이언트 컴포넌트만 다시 렌더링
// 서버 컴포넌트는 재실행되지 않음 → 서버 데이터 오류에는 효과 없음
<button onClick={() => reset()}>재시도</button>
```

**2. `window.location.reload()` — 전체 페이지 새로고침**

```typescript
// 모든 상태 초기화 + 서버 컴포넌트 포함 전체 재실행
// 가장 확실하지만 클라이언트 상태 손실
<button onClick={() => window.location.reload()}>새로고침</button>
```

**3. `router.refresh()` + `startTransition` — 서버 컴포넌트 재실행**

```typescript
'use client';
import { useRouter } from 'next/navigation';
import { startTransition } from 'react';

export default function Error({ error, reset }: { error: Error; reset: () => void }) {
  const router = useRouter();

  return (
    <button
      onClick={() => {
        // 서버 컴포넌트를 재실행하고, 성공하면 Error Boundary도 초기화
        startTransition(() => {
          router.refresh();
          reset();
        });
      }}
    >
      재시도
    </button>
  );
}
```

### 전역 에러 처리

```
app/
  error.tsx        ← 각 라우트별 에러 처리
  global-error.tsx ← 루트 레이아웃 에러까지 처리 (최상위 fallback)
```

## Pitfalls
- `error.tsx`는 **같은 디렉토리의 `layout.tsx`에서 발생한 에러는 잡지 못한다.** 레이아웃 에러는 상위 디렉토리의 `error.tsx`가 처리한다.
- `reset()`만 호출하면 서버 컴포넌트가 재실행되지 않으므로, 서버에서 데이터 패칭 실패로 발생한 에러는 `reset()`만으로 해결되지 않는다. `router.refresh()`와 함께 써야 한다.
- `global-error.tsx`는 루트 레이아웃까지 대체하므로 `<html>`, `<body>` 태그를 직접 포함해야 한다.
