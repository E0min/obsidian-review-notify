# 재검증하기 (revalidatePath / revalidateTag)
> TL;DR: 빌드 타임 캐시를 특정 경로 또는 데이터 태그 단위로 무효화해 다음 요청 시 서버가 최신 데이터로 페이지를 재생성하게 만드는 On-demand Revalidation.

## What
Next.js의 캐시(데이터 캐시·풀 라우트 캐시)는 기본적으로 빌드 타임에 생성되어 재배포 전까지 유지된다. `revalidatePath`와 `revalidateTag`는 배포 없이도 특정 캐시를 즉시 무효화(PURGE)할 수 있는 함수다. 서버 액션이나 Route Handler에서 호출한다.

- `revalidatePath`: 특정 URL 경로의 캐시 무효화
- `revalidateTag`: fetch 태그로 연결된 데이터 캐시만 선택적 무효화

## Why it matters
- 콘텐츠 수정 시 재배포 없이 캐시를 즉시 무효화할 수 있다.
- `revalidateTag`를 사용하면 **페이지 전체 재생성 없이** 변경된 데이터만 갱신해 성능을 최적화한다.
- CMS, 블로그, 쇼핑몰처럼 콘텐츠가 자주 바뀌는 서비스에서 ISR과 함께 핵심 패턴으로 쓰인다.

## How

### `revalidatePath` — 경로 기반 무효화

```typescript
// app/actions.ts
'use server';
import { revalidatePath } from 'next/cache';

// 특정 페이지만 무효화
revalidatePath('/book/1');

// 동적 경로의 모든 페이지 무효화 (e.g. /book/1, /book/2, ...)
revalidatePath('/book/[id]', 'page');

// 해당 레이아웃을 공유하는 모든 페이지 무효화
revalidatePath('/layout', 'layout');

// 전체 사이트 무효화 (루트 레이아웃 기준)
revalidatePath('/', 'layout');
```

### `revalidateTag` — 태그 기반 무효화

fetch 시 태그를 붙여두고, 필요할 때 해당 태그만 무효화한다.

```typescript
// 데이터 패칭 시 태그 지정
const reviews = await fetch('https://api.example.com/reviews/1', {
  next: { tags: ['review-1'] },
}).then(r => r.json());

// 리뷰 수정 서버 액션
export async function updateReview(reviewId: string) {
  // DB 수정 로직...
  // review-1 태그가 붙은 fetch 캐시만 무효화
  revalidateTag(`review-1`);
}
```

### 캐시 동작 사이클

```
PURGE (revalidatePath / revalidateTag 호출)
  ↓
STALE (캐시 무효화 상태)
  ↓
다음 요청 발생
  ↓
FETCH (서버에서 새로 데이터 가져옴)
  ↓
SET (새 데이터로 캐시 갱신)
  ↓
HIT (이후 요청은 새 캐시에서 반환)
```

### 실전 패턴 — 서버 액션과 조합

```typescript
// app/actions.ts
'use server';
import { revalidatePath } from 'next/cache';

export async function deleteBook(bookId: string) {
  await db.book.delete({ where: { id: bookId } });
  // 삭제 후 도서 목록 페이지와 해당 도서 상세 페이지 캐시 무효화
  revalidatePath('/books');
  revalidatePath(`/book/${bookId}`);
}
```

## Pitfalls
- `revalidatePath`와 `revalidateTag`는 **서버 컴포넌트 또는 서버 액션에서만 호출 가능**하다. 클라이언트 컴포넌트에서 직접 호출하면 에러가 발생한다.
- 재검증 후 **첫 요청에서 풀 라우트 캐시가 즉시 SET되지 않는다**. 캐시를 PURGE하면 STALE 상태가 되고, 실제 사용자가 해당 페이지를 방문하는 시점에 서버가 렌더링해 캐시에 저장한다. 따라서 재검증 직후 첫 방문자는 약간 느릴 수 있다.
- `revalidatePath('/', 'layout')`은 모든 페이지를 무효화하므로 트래픽이 많은 서비스에서 남용하면 서버 부하가 급증한다. 가능한 범위를 좁혀서 사용하는 것이 좋다.
