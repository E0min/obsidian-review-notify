# 타입 좁히기 (Type Narrowing)

> TL;DR: 유니온 타입(`string | number`)을 조건문 안에서 더 구체적인 타입으로 좁히는 과정. TypeScript 컴파일러가 제어 흐름을 분석해 자동 추론한다.

---

## What

`string | number | null` 같은 넓은 타입에서 특정 분기 안에서만 확실히 알 수 있는 구체적인 타입으로 컴파일러가 추론 범위를 좁히는 것.

```typescript
function printId(id: string | number) {
  // 여기서 id: string | number
  if (typeof id === 'string') {
    // 여기서 id: string — 좁혀짐
    console.log(id.toUpperCase());
  } else {
    // 여기서 id: number
    console.log(id.toFixed(2));
  }
}
```

---

## How — 6가지 narrowing 방법

### 1. `typeof` 가드 (원시 타입)

```typescript
function processValue(val: string | number | boolean | null) {
  if (typeof val === 'string') {
    return val.toUpperCase(); // string
  }
  if (typeof val === 'number') {
    return val * 2;           // number
  }
  if (typeof val === 'boolean') {
    return !val;              // boolean
  }
  return null;               // null (typeof null === 'object' 주의)
}
```

**주의**: `typeof null === 'object'` — null 체크는 `typeof`보다 `=== null` 사용

### 2. `instanceof` 가드 (클래스 인스턴스)

```typescript
class Dog { bark() { return '멍'; } }
class Cat { meow() { return '야옹'; } }

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    return animal.bark(); // Dog
  }
  return animal.meow();   // Cat
}
```

### 3. Truthiness 가드 (falsy 제거)

```typescript
function printLength(val: string | null | undefined) {
  if (val) {
    // val: string (null/undefined/'' 제거됨)
    console.log(val.length);
  }
}

// 주의: ''(빈 문자열)도 falsy → 의도치 않게 걸러질 수 있음
// 더 명확한 패턴:
if (val != null) { ... } // null + undefined만 제거
```

### 4. 동등 비교 (`==`, `===`, `switch`)

```typescript
type Status = 'loading' | 'success' | 'error';

function handleStatus(status: Status, data: string | null) {
  if (data === null) {
    // data: null
    return '데이터 없음';
  }
  // data: string
  switch (status) {
    case 'loading': return '로딩 중...';
    case 'success': return data.toUpperCase();
    case 'error':   return '오류 발생';
  }
}
```

### 5. `in` 연산자 가드 (프로퍼티 존재 여부)

```typescript
interface Fish { swim: () => void }
interface Bird { fly: () => void }

function move(animal: Fish | Bird) {
  if ('swim' in animal) {
    animal.swim(); // Fish
  } else {
    animal.fly();  // Bird
  }
}
```

### 6. 판별 유니온 (Discriminated Union) — 가장 강력

태그 필드(`kind`, `type`)로 타입을 구별하는 패턴.

```typescript
interface LoadingState  { status: 'loading' }
interface SuccessState  { status: 'success'; data: string }
interface ErrorState    { status: 'error';   message: string }

type AsyncState = LoadingState | SuccessState | ErrorState;

function render(state: AsyncState) {
  switch (state.status) {
    case 'loading': return <Spinner />;
    case 'success': return <div>{state.data}</div>;    // data 접근 가능
    case 'error':   return <div>{state.message}</div>; // message 접근 가능
  }
}
```

### 사용자 정의 타입 가드 (`is`)

```typescript
function isString(val: unknown): val is string {
  return typeof val === 'string';
}

function processUnknown(val: unknown) {
  if (isString(val)) {
    // val: string
    console.log(val.toUpperCase());
  }
}
```

`is` 키워드로 반환 타입을 `boolean`이 아닌 타입 서술어로 지정 → 컴파일러에게 좁혀진 타입 전달.

---

## Why it matters

```typescript
// ❌ narrowing 없이 접근 시 컴파일 에러
function badExample(val: string | number) {
  val.toUpperCase(); // 에러: Property 'toUpperCase' does not exist on type 'number'
}

// ✅ narrowing 후 안전하게 접근
function goodExample(val: string | number) {
  if (typeof val === 'string') {
    val.toUpperCase(); // OK
  }
}
```

---

## Pitfalls

- **`typeof null === 'object'`**: null 체크는 `val === null` 명시적으로
- **빈 문자열 falsy**: `if (val)` 패턴은 `''`도 제외 → `if (val != null)` 또는 `if (val !== null && val !== undefined)` 구분
- **`instanceof`는 클래스에만**: 인터페이스(`interface`)에는 사용 불가 → `in` 연산자 또는 판별 유니온 사용
- **narrowing 후 재할당 금지**: narrowing된 블록 안에서 변수를 넓은 타입으로 재할당하면 타입이 다시 넓어짐
