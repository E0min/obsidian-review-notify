# 이벤트 처리하기 (Event Handling)

> TL;DR: React 이벤트는 camelCase + 함수 참조 전달 방식. `onClick={fn}` (O), `onClick={fn()}` (X). SyntheticEvent로 크로스 브라우저 통일 처리.

## What

React에서 DOM 이벤트에 핸들러 함수를 연결하는 방법. HTML과 달리 camelCase + 함수 참조 전달.

## Why it matters

이벤트 처리는 사용자 인터랙션의 기본. 함수 참조와 즉시 실행의 차이를 혼동하면 무한 렌더링 등 버그가 발생한다.

## How

**기본 구조**:
```javascript
function MyButton() {
    function handleClick() { alert('클릭!'); }
    return <button onClick={handleClick}>Click</button>;
}
```

**주의**: `onClick={handleClick()}` (X) → 즉시 실행됨. `onClick={handleClick}` (O) → 함수 참조

**이벤트 객체** (SyntheticEvent — 크로스 브라우저 통일):
```javascript
function handleChange(event) {
    console.log(event.target.value);
}
<input onChange={handleChange} />
```

**인자 전달**:
```javascript
<button onClick={() => handleClick(1)}>Click</button>
```

**기본 동작 방지**:
```javascript
function handleSubmit(event) {
    event.preventDefault();
}
```

**이벤트 전파 방지**:
```javascript
function handleChildClick(event) {
    event.stopPropagation();
}
```

## Pitfalls

- HTML: `onclick="fn()"` vs React: `onClick={fn}` (카멜케이스, 함수 참조)
- 클래스형 컴포넌트에서 this 바인딩 필요 (함수형에선 불필요)
