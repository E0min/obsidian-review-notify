# 커스텀 훅 (Custom Hook)

> TL;DR: `use`로 시작하는 사용자 정의 훅. 반복되는 상태 로직을 함수로 분리해 여러 컴포넌트에서 재사용.

## What

`useState`, `useEffect` 등을 조합해 반복 로직을 모듈화한 사용자 정의 훅. 이름은 반드시 `use`로 시작.

## Why it matters

여러 컴포넌트에 반복되는 로직(API 호출, 폼 관리 등)을 한 곳에서 관리 → 중복 제거 + 유지보수 향상.

## How

```javascript
// useFetch.js
import { useState, useEffect } from 'react';

function useFetch(url) {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    const [error, setError] = useState(null);

    useEffect(() => {
        setLoading(true);
        fetch(url)
            .then(res => res.json())
            .then(data => { setData(data); setLoading(false); })
            .catch(err => { setError(err); setLoading(false); });
    }, [url]);

    return { data, loading, error };
}

// 컴포넌트에서 사용
const { data, loading, error } = useFetch('/api/posts');
```

## 커스텀 훅 vs 다른 방식

- **vs 일반 함수**: 커스텀 훅은 useState/useEffect 등 훅 사용 가능
- **vs HOC**: 커스텀 훅이 더 간결하고 가독성 좋음
- **vs Render Props**: 중첩 없이 훅으로 로직 분리

## Pitfalls

- 이름 `use`로 시작 필수 (Lint 규칙)
- 컴포넌트 최상위에서만 호출 (조건문/루프 내 불가)
