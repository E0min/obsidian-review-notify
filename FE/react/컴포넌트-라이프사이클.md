# 컴포넌트의 라이프 사이클

> TL;DR: 컴포넌트가 생성(마운트) → 업데이트 → 소멸(언마운트)되는 3단계. 함수형에서는 useEffect로 모두 처리.

## What

컴포넌트가 생성→업데이트→소멸되는 3단계 과정.

## Why it matters

라이프사이클을 이해해야 API 호출 시점, 이벤트 리스너 정리, 구독 해제 등을 올바르게 처리할 수 있음.

## How

| 단계 | 클래스형 | 함수형 |
|------|---------|--------|
| **마운트** | componentDidMount | useEffect(fn, []) |
| **업데이트** | componentDidUpdate | useEffect(fn, [dep]) |
| **언마운트** | componentWillUnmount | useEffect(() => () => cleanup, []) |

```javascript
// 함수형 컴포넌트 예시
useEffect(() => {
    console.log('마운트');
    return () => console.log('언마운트');
}, []);

useEffect(() => {
    console.log('count 변경:', count);
}, [count]);
```

## Pitfalls

- cleanup 함수는 반드시 함수를 반환해야 함 (문자열 등 다른 값 반환 시 에러)
