# React 라우팅 (react-router-dom)

> TL;DR: SPA에서 URL에 따라 다른 컴포넌트를 렌더링하는 라이브러리. `<Link>`로 이동하고, `useNavigate` / `useParams` / `useSearchParams`로 라우팅 정보를 활용한다.

## What

SPA에서 URL에 따라 다른 컴포넌트를 렌더링하는 라이브러리.

## Why it matters

React 자체에는 라우팅 기능이 없다. `react-router-dom`이 URL 변화를 감지하여 페이지 새로고침 없이 컴포넌트를 교체해주므로, SPA에서 다중 페이지 경험을 구현할 수 있다.

## How

```shell
npm i react-router-dom
```

**기본 구조**:

```javascript
import { BrowserRouter, Routes, Route, Link, useNavigate, useParams } from 'react-router-dom';

function App() {
    return (
        <BrowserRouter>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/about" element={<About />} />
                <Route path="/user/:id" element={<User />} />
            </Routes>
        </BrowserRouter>
    );
}
```

**주요 훅**:

```javascript
// 프로그래밍 방식 이동
const navigate = useNavigate();
navigate('/about');

// URL 파라미터
const { id } = useParams(); // /user/123 → id = "123"

// 쿼리 스트링
const [searchParams] = useSearchParams();
const query = searchParams.get('query'); // ?query=react → "react"

// 현재 위치
const location = useLocation();
```

**중첩 라우트** + `Outlet`:

```javascript
<Route path="/" element={<Layout />}>
    <Route index element={<Home />} />
    <Route path="about" element={<About />} />
</Route>
// Layout에서 <Outlet /> 위치에 자식 렌더링
```

**URL 파라미터 vs 쿼리 스트링**:

| | URL 파라미터 | 쿼리 스트링 |
|--|------------|-----------|
| 형태 | `/user/:id` | `/search?query=react` |
| 용도 | 리소스 식별 | 검색 / 필터링 |
| 필수 여부 | 필수 | 선택적 |

## Pitfalls

- `<a href>` 태그 사용 시 전체 페이지 새로고침 발생 → 반드시 `<Link to>` 사용
- `<Routes>` 내부에는 `<Route>`만 직접 자식으로 배치
- `BrowserRouter`는 최상위에 한 번만 감싸면 됨 (중첩 금지)
