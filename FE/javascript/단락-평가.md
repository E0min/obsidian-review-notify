# 단락 평가 (Short-circuit Evaluation)

> TL;DR: `&&`와 `||` 연산자는 좌항의 Truthy/Falsy에 따라 우항 평가를 건너뛰며, 이를 활용해 기본값 설정과 조건부 실행을 간결하게 쓸 수 있다.

## What

논리 연산자 `&&`, `||`는 불린을 반환하는 게 아니라, **실제로 평가된 피연산자의 값을 반환**한다.

| 연산자 | 동작 |
|--------|------|
| `expr1 && expr2` | `expr1`이 **falsy**면 `expr1` 반환 (우항 평가 안 함) |
| `expr1 && expr2` | `expr1`이 **truthy**면 `expr2` 반환 |
| `expr1 \|\| expr2` | `expr1`이 **truthy**면 `expr1` 반환 (우항 평가 안 함) |
| `expr1 \|\| expr2` | `expr1`이 **falsy**면 `expr2` 반환 |

## Why it matters

조건문 없이 기본값 설정, 조건부 실행, 안전한 속성 접근을 한 줄로 표현할 수 있어 코드가 간결해진다.

## How

### 활용 1 — 기본값 설정

```javascript
function greet(name) {
    name = name || '익명 사용자';
    console.log('안녕하세요, ' + name + '!');
}

greet();          // 안녕하세요, 익명 사용자!
greet('영민');    // 안녕하세요, 영민!
```

### 활용 2 — 조건부 실행

```javascript
let loggedIn = true;
loggedIn && console.log('사용자가 로그인했습니다.');
// loggedIn이 falsy면 console.log 자체가 실행되지 않음
```

### 활용 3 — 함수 호출 제어

```javascript
// user가 null이면 logout()을 호출하지 않아 에러 방지
user && user.logout();
```

### 활용 4 — 중첩 속성 안전 접근

```javascript
// person.details가 없으면 .age 접근 시 TypeError 발생을 방지
let age = person.details && person.details.age;
```

> 참고: ES2020 이후는 옵셔널 체이닝 `person.details?.age`를 쓰는 게 더 명확하다.

## Pitfalls

- `||`는 `0`, `""`, `false` 같은 **유효한 falsy 값**도 기본값으로 대체해 버린다. 이럴 땐 `??` (nullish coalescing)를 사용해야 한다.

```javascript
// 문제: count가 0이면 의도치 않게 10이 할당됨
let count = inputCount || 10;

// 해결: null/undefined일 때만 기본값 적용
let count = inputCount ?? 10;
```

- `&&` 체인이 너무 길어지면 가독성이 떨어진다. 복잡한 조건은 `if` 문이 낫다.
