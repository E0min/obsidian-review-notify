# 동기와 비동기

> TL;DR: 동기는 순차 실행, 비동기는 완료를 기다리지 않고 다음 코드를 실행하며, Promise는 비동기 작업의 성공/실패를 표준화된 방식으로 다룬다.

## What

- **동기(Synchronous)**: 코드가 위에서 아래로 순차 실행. 앞 작업이 완료되어야 다음 실행 가능.
- **비동기(Asynchronous)**: 특정 작업이 완료될 때까지 기다리지 않고 다음 코드를 먼저 실행. 완료 시 콜백으로 결과를 처리.

## Why it matters

네트워크 요청, 파일 읽기, 타이머 등 시간이 걸리는 작업을 동기로 처리하면 브라우저가 멈춘다(블로킹). 비동기를 이해해야 이런 작업을 효율적으로 다룰 수 있다.

## How

### 동기 vs 비동기 실행 순서 비교

```javascript
// 동기 — 순서대로 실행
console.log("1번!");
console.log("2번!");
console.log("3번!");
// 출력: 1번! → 2번! → 3번!

// 비동기 — setTimeout은 타이머 완료 후 실행
setTimeout(() => console.log("1번!"), 3000);
console.log("2번!");
console.log("3번!");
// 출력: 2번! → 3번! → (3초 후) 1번!
```

### 콜백 지옥 (Callback Hell)

비동기 작업을 콜백으로 중첩하면 가독성이 파괴된다.

```javascript
// 중첩이 깊어질수록 유지보수 불가
doStep1(function(result1) {
    doStep2(result1, function(result2) {
        doStep3(result2, function(result3) {
            doStep4(result3, function(result4) {
                console.log('완료:', result4);
            });
        });
    });
});
```

### Promise — 비동기 작업의 표준화

Promise는 3가지 상태를 가진다.

| 상태 | 설명 |
|------|------|
| **Pending** (대기) | 비동기 작업 진행 중 |
| **Fulfilled** (이행) | 성공, `resolve()` 호출됨 |
| **Rejected** (거부) | 실패, `reject()` 호출됨 |

```javascript
// Promise 생성
const promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        const success = true;
        if (success) {
            resolve("작업 성공!");  // Fulfilled → .then() 실행
        } else {
            reject("작업 실패!");   // Rejected → .catch() 실행
        }
    }, 500);
});

// Promise 사용
promise
    .then(result => console.log(result))   // "작업 성공!"
    .catch(error => console.log(error));   // 에러 처리
```

### Promise 체이닝으로 콜백 지옥 해결

```javascript
doStep1()
    .then(result1 => doStep2(result1))
    .then(result2 => doStep3(result2))
    .then(result3 => doStep4(result3))
    .then(result4 => console.log('완료:', result4))
    .catch(err => console.error('에러:', err));
```

## Pitfalls

- `setState` (React), `localStorage.setItem` 처럼 **동기처럼 보이지만 비동기**인 함수들을 주의해야 한다.
- Promise 체이닝에서 `.catch()`를 누락하면 **에러가 조용히 삼켜진다** (unhandled rejection).
- `new Promise` 내부에서 `throw`를 던지면 자동으로 `reject`로 처리되지만, 비동기 콜백 안에서 `throw`는 Promise가 잡지 못한다.

```javascript
// catch 없으면 에러 감지 불가
fetch('/api/data')
    .then(res => res.json());
    // .catch() 빠짐 → 네트워크 에러 시 조용히 실패

// 올바른 패턴
fetch('/api/data')
    .then(res => res.json())
    .catch(err => console.error('요청 실패:', err));
```
