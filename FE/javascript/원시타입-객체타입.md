# 원시 타입과 객체 타입

> TL;DR: 원시 타입은 값 자체를 스택에 저장하고, 객체 타입은 힙의 참조 주소를 저장하기 때문에 비교·복사 동작이 완전히 다르다.

## What

### 원시 타입 (Primitive Type) — 7가지

`Boolean`, `Null`, `Undefined`, `Number`, `BigInt`, `String`, `Symbol`

- 값 자체가 스택(Stack) 메모리에 저장됨
- **불변(immutable)**: 값을 변경하면 새 값이 생성됨
- `===` 비교 시 **값**을 비교

### 객체 타입 (Object Type)

`Object`, `Array`, `Function`, `Date`, `RegExp`, `Map`, `Set` 등

- 힙(Heap) 메모리에 데이터를 저장하고, 스택에는 **참조(주소)**만 저장됨
- **변경 가능(mutable)**
- `===` 비교 시 **참조(주소)**를 비교

## Why it matters

참조 비교 특성을 모르면 예상치 못한 버그가 생긴다. React의 상태 불변성 원칙도 이 개념에서 비롯된다.

## How

### 원시 타입 복사 — 값이 복사됨

```javascript
let a = 10;
let b = a;   // b에 10이라는 값 자체가 복사됨
b = 20;

console.log(a); // 10 — a는 영향 없음
console.log(b); // 20
```

### 객체 타입 비교 — 참조 비교

```javascript
let object1 = { name: 'John', age: 30 };
let object2 = { name: 'John', age: 30 };
let object3 = object1;

console.log(object1 === object2); // false — 내용이 같아도 다른 주소
console.log(object1 === object3); // true  — 같은 주소를 가리킴
```

### 객체 타입 복사 — 얕은 복사의 함정

```javascript
let original = { name: 'Kim', age: 25 };
let copy = original; // 주소 복사! 같은 객체를 가리킴

copy.name = 'Lee';
console.log(original.name); // 'Lee' — 원본도 변경됨
```

### 내용 비교 방법

```javascript
// 방법 1: JSON.stringify (중첩 객체 비교 가능, 함수·undefined 제외됨)
JSON.stringify(object1) === JSON.stringify(object2);

// 방법 2: 직접 비교 함수 작성
function isEqual(obj1, obj2) {
    return obj1.name === obj2.name && obj1.age === obj2.age;
}
```

## Pitfalls

- 배열도 객체 타입이므로 `[] === []`는 `false`다.
- `const`로 선언된 객체도 내부 속성은 변경 가능하다. `const`는 재할당만 막는다.

```javascript
const obj = { a: 1 };
obj.a = 2;    // 가능 — 내부 속성 변경
obj = { b: 3 }; // TypeError — 재할당 불가
```
