# 배열 관련 메소드

> TL;DR: 배열 메소드는 추가/삭제, 탐색, 변형 세 가지 범주로 나뉘며, `map·filter·reduce`는 원본을 바꾸지 않는 함수형 패턴의 핵심이다.

## What

자바스크립트 배열이 기본 제공하는 내장 메소드 모음. 범주별로 구분하면 기억하기 쉽다.

## Why it matters

올바른 메소드 선택이 코드 간결성과 의도 전달에 큰 차이를 만든다. 특히 원본 배열 변경 여부를 파악하지 않으면 사이드 이펙트가 발생한다.

## How

### 추가 / 삭제

```javascript
const arr = [1, 2, 3];

arr.push(4);          // 끝에 추가 → [1, 2, 3, 4]
arr.pop();            // 끝 제거 및 반환 → 4
arr.unshift(0);       // 앞에 추가 → [0, 1, 2, 3]
arr.shift();          // 앞 제거 및 반환 → 0

arr.slice(1, 3);      // 인덱스 1~2 복사 추출 (원본 변경 없음) → [2, 3]
arr.concat([4, 5]);   // 배열 결합, 새 배열 반환 → [1, 2, 3, 4, 5]
```

### 순회

```javascript
// forEach — 각 요소에 함수 실행, 반환값 없음(undefined), break 불가
[1, 2, 3].forEach((item, index) => {
    console.log(index, item);
});
```

### 탐색

```javascript
const fruits = ['apple', 'banana', 'cherry'];

fruits.includes('banana');             // true — 포함 여부 (boolean)
fruits.indexOf('banana');              // 1 — 첫 인덱스, 없으면 -1
fruits.findIndex(f => f.length > 5);  // 1 — 조건 충족 첫 인덱스
fruits.find(f => f.length > 5);       // 'banana' — 조건 충족 첫 요소 값
```

### 필터

```javascript
const numbers = [1, 2, 3, 4, 5];

// 조건 충족 요소들로 새 배열 (원본 변경 없음)
numbers.filter(n => n % 2 === 0); // [2, 4]
```

### 변형

```javascript
// join — 구분자로 문자열 결합
['a', 'b', 'c'].join('-'); // 'a-b-c'

// map — 각 요소 변환, 새 배열 반환 (원본 변경 없음)
[1, 2, 3].map(n => n * 2); // [2, 4, 6]

// sort — 배열 정렬 (원본 직접 변경!)
['banana', 'apple', 'cherry'].sort(); // ['apple', 'banana', 'cherry']
[4, 2, 5, 1].sort((a, b) => a - b);  // [1, 2, 4, 5] — 숫자는 비교함수 필수

// reduce — 누산, 하나의 값으로 합침
[1, 2, 3, 4].reduce((acc, n) => acc + n, 0); // 10
```

---

### `map` 상세 — "새로운 배열? 객체?"

**결론: 항상 새로운 배열(Array)을 반환한다.** 원본 배열은 절대 변경되지 않음.

#### 직관적 이해 — "데이터 가공 공장"

`map()`은 **기존 배열의 데이터를 하나씩 꺼내서 가공한 뒤, 완전히 새로운 배열을 만들어내는 공장**이다. 원본 배열은 절대 건드리지 않는다.

> `new Map()`(키-값 보관함 자료구조)과는 이름만 같고 완전히 다른 역할 — [[Map-객체]] 참고.

```javascript
const 새배열 = 기존배열.map((현재값, 인덱스) => {
    return 가공된_새로운_값;
    // 현재값: 배열에서 꺼낸 요소 (필수)
    // 인덱스: 현재 요소의 위치, 0부터 시작 (선택)
});
```

**실무 패턴 1 — 숫자·문자 일괄 변환**

```javascript
const numbers = [1, 2, 3, 4];
const doubled = numbers.map(num => num * 2); // 화살표 함수: return 생략 가능

console.log(doubled); // [2, 4, 6, 8] ← 새 배열 탄생
console.log(numbers); // [1, 2, 3, 4] ← 원본은 안전
```

**실무 패턴 2 — 객체 배열에서 특정 값만 추출** (API 응답 가공 시 가장 많이 쓰임)

```javascript
const users = [
    { id: 1, name: "김철수", age: 25 },
    { id: 2, name: "이영희", age: 28 },
    { id: 3, name: "박개발", age: 30 },
];

const namesOnly = users.map(user => user.name);
console.log(namesOnly); // ["김철수", "이영희", "박개발"]
```

**가장 흔한 실수 — `return` 누락**

```javascript
const numbers = [1, 2, 3];

// ❌ 중괄호 블록 쓰면 return 명시 필수
const result = numbers.map(num => {
    num * 2; // return 없음 → undefined 반환
});
console.log(result); // [undefined, undefined, undefined]

// ✅ 중괄호 없으면 암묵적 return (화살표 함수 단축 표현)
const result2 = numbers.map(num => num * 2);
console.log(result2); // [2, 4, 6]
```

> `forEach()`는 배열을 순회하며 부수 효과(로그, DOM 조작)만 할 때. `map()`은 **새로운 배열 결과물이 필요할 때** — 반환값 유무가 결정적 차이.

---

```
원본 배열 [a, b, c]
       │  각 요소마다 콜백 실행
       ▼
  callback(a) → x
  callback(b) → y   →  새 배열 [x, y, z]
  callback(c) → z
```

- 반환되는 배열의 **길이는 원본과 항상 동일**
- 콜백이 반환하는 값이 새 배열의 각 요소가 됨 (숫자, 문자열, 객체, 배열 모두 가능)

#### 콜백 인자

```javascript
array.map((element, index, array) => { ... })
//         현재 요소  인덱스  원본 배열(선택)
```

#### 반환값 타입별 예시

```javascript
// 숫자 → 숫자
[1, 2, 3].map(n => n * 2);           // [2, 4, 6]

// 문자열 → 문자열
['hello', 'world'].map(s => s.toUpperCase()); // ['HELLO', 'WORLD']

// 배열 → 객체  (콜백이 객체를 반환해도 결과는 배열)
const users = [
  { name: '영민', age: 28 },
  { name: '지수', age: 24 },
];
users.map(u => u.name);          // ['영민', '지수']  ← 배열
users.map(u => ({ ...u, adult: u.age >= 18 }));
// [{ name: '영민', age: 28, adult: true }, { name: '지수', age: 24, adult: true }]
// 각 요소가 객체이지만, 전체 반환값은 배열!
```

#### `map` vs `forEach` — 핵심 차이

| | `map` | `forEach` |
|--|-------|-----------|
| 반환값 | **새 배열** | `undefined` |
| 사용 목적 | 변환 결과가 필요할 때 | 부수 효과(로그, DOM 조작 등) |
| 체이닝 | `.map().filter()` 가능 | 체이닝 불가 |

```javascript
// map — 결과를 담아서 쓸 때
const doubled = [1, 2, 3].map(n => n * 2); // [2, 4, 6]

// forEach — 결과가 필요 없을 때
[1, 2, 3].forEach(n => console.log(n));     // undefined 반환, 담을 수 없음
```

> **주의**: `map` 안에서 `return`을 빼먹으면 `undefined`로 가득 찬 배열이 나온다.
> ```javascript
> [1, 2, 3].map(n => { n * 2 }); // [undefined, undefined, undefined] ← return 없음!
> [1, 2, 3].map(n => n * 2);     // [2, 4, 6] ← 화살표 함수 암묵적 return
> ```

#### `map` 내부 동작 — myMap으로 이해하기

V8 엔진이 C++로 구현한 `map`을 자바스크립트로 표현하면 구조가 명확해진다.

```javascript
Array.prototype.myMap = function(callback) {
  // 1. 완전히 새로운 빈 배열 생성
  const newArray = [];

  // 2. 원본 배열(this) 길이만큼 반복
  for (let i = 0; i < this.length; i++) {

    // 3. 콜백 실행 — 현재 값·인덱스·원본 배열을 인자로 전달
    const returnedValue = callback(this[i], i, this);

    // 4. 콜백이 return한 값을 새 배열에 push
    newArray.push(returnedValue);
  }

  // 5. 완성된 새 배열 반환
  return newArray;
};
```

**`return`이 필수인 이유**: 3번 단계에서 콜백이 아무것도 `return`하지 않으면 자바스크립트는 자동으로 `undefined`를 반환한다. 그러면 4번 단계에서 `newArray.push(undefined)`가 실행되어 `undefined`로 가득 찬 배열이 완성된다.

```javascript
// ❌ return 없음 → undefined push
[1, 2, 3].myMap(n => { n * 2 }); // [undefined, undefined, undefined]

// ✅ return 있음 → 결과값 push
[1, 2, 3].myMap(n => n * 2);     // [2, 4, 6]
```

**핵심 멘탈 모델**: `map`은 기존 배열을 **설계도**로 삼아, 각 요소를 콜백에 찍어내어 **결과값만 모아 새 배열을 조립**하는 공정이다. 원본 배열은 변형되지 않는다.

#### `Map 객체`와 헷갈리지 않기

| | `Array.map()` | `new Map()` |
|--|--------------|-------------|
| 종류 | **배열 메서드** | **자료구조** |
| 결과 | 새 배열 반환 | 키-값 저장 컬렉션 |
| 사용처 | 배열 변환 | 데이터 저장·탐색 |

### 자주 쓰는 조합 패턴

```javascript
const users = [
    { name: '영민', age: 28, active: true },
    { name: '지수', age: 24, active: false },
    { name: '민준', age: 31, active: true },
];

// 활성 유저의 이름만 뽑기
users
    .filter(u => u.active)
    .map(u => u.name);
// ['영민', '민준']

// 전체 나이 합계
users.reduce((sum, u) => sum + u.age, 0); // 83
```

## Pitfalls

- `sort()`는 **원본 배열을 직접 변경**한다. 원본 보존이 필요하면 `[...arr].sort()` 사용.
- 숫자 배열에 비교 함수 없이 `sort()`를 쓰면 유니코드 순서로 정렬된다.

```javascript
[10, 9, 2].sort();            // [10, 2, 9] — 잘못된 결과!
[10, 9, 2].sort((a, b) => a - b); // [2, 9, 10] — 올바른 결과
```

- `find`와 `findIndex`는 조건을 만족하는 **첫 번째** 요소만 반환한다. 모든 요소를 원하면 `filter` 사용.
