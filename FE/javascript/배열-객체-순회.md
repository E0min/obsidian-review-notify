# 배열과 객체의 순회 탐색 방법

> TL;DR: 배열은 `for` / `forEach` / `for...of`, 객체는 `for...in` / `Object.keys·values·entries()`로 순회하며, 각각 break 가능 여부와 비동기 지원 여부가 다르다.

## What

배열과 객체를 순회하는 다양한 방법과 각각의 적합한 상황.

## Why it matters

올바른 순회 방법 선택이 코드 가독성과 성능에 직결된다. 특히 `break` 필요 여부, 비동기 처리 여부에 따라 선택이 달라진다.

## How

### 배열 순회

**1. `for` 루프** — 인덱스 제어 필요, `break`/`continue` 가능

```javascript
for (let i = 0; i < array.length; i++) {
    console.log(array[i]);
}
```

**2. `forEach()`** — 부수 효과(side effect) 목적, `break` 불가

```javascript
array.forEach(function(value, index) {
    console.log(index, value);
});
```

**3. `for...of`** — ES6+, 값 중심 순회, `break`/`continue` 가능

```javascript
for (let value of array) {
    console.log(value);
}
```

---

### `for` vs `forEach` 핵심 차이

| 항목 | `for` | `forEach` |
|------|-------|-----------|
| `break` / `continue` | ✅ 가능 | ❌ 불가 |
| 인덱스 접근 | `i` 변수로 직접 | 콜백 두 번째 인자 |
| 반환값 | 없음 | `undefined` |
| async/await | ✅ 가능 | ❌ 비동기 미지원 |
| 빈 요소 처리 | 빈 슬롯 포함 | 빈 슬롯 건너뜀 |
| 성능 | 약간 빠름 | 약간 느림 (함수 호출 오버헤드) |

---

### 상황별 선택 가이드

```javascript
// break가 필요한 경우 → for 사용
for (let i = 0; i < arr.length; i++) {
    if (arr[i] === target) break;
}

// 단순 순회, 가독성 중심 → forEach 사용
arr.forEach(item => console.log(item));

// async 작업이 필요한 경우 → for...of 사용
for (const item of arr) {
    await processItem(item);
}
```

---

### 객체 순회

**1. `for...in`** — 키 순회 (상속된 열거 가능 속성 포함되므로 주의)

```javascript
const person = { name: '영민', age: 28 };

for (let key in person) {
    console.log(key + ': ' + person[key]);
}
// name: 영민
// age: 28
```

**2. `Object.keys / values / entries()`** — 자신의 속성만, 배열로 반환

```javascript
// 키 목록
Object.keys(person).forEach(key => console.log(key));

// 값 목록
Object.values(person).forEach(value => console.log(value));

// [키, 값] 쌍 목록
Object.entries(person).forEach(([key, value]) => {
    console.log(key, value);
});
```

## Pitfalls

- `forEach`에서 `break`를 쓰려면 `for...of`나 `for`로 전환해야 한다.
- `for...in`은 **배열에 사용 금지** — 프로토타입 체인의 속성까지 순회할 수 있고, 인덱스가 문자열로 반환된다.
- `forEach`는 비동기 처리 시 `await`가 정상 동작하지 않는다 → `for...of` 사용.

```javascript
// 잘못된 패턴: forEach 내부 await는 병렬 실행됨 (순서 보장 X)
arr.forEach(async (item) => {
    await processItem(item); // 기대와 다르게 동작
});

// 올바른 패턴: for...of 로 순차 비동기 처리
for (const item of arr) {
    await processItem(item);
}
```
