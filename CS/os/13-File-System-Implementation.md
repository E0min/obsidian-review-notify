# 13. File-System Implementation

> TL;DR: 파일 시스템 구현 - 계층적 구조(입출력 제어/기본/파일구성/논리), FCB(inode), 디렉터리 구현(선형 리스트/해시), 할당 방법(연속/연결/색인), 자유공간 관리.

---

## 1. File-System Structure
- 메모리와 디스크 간의 입출력 전송은 **블록 단위**로 수행됨. 각 블록은 하나 이상의 섹터 (통상 512바이트)

**파일 시스템의 계층적 구조**:

| 계층 | 설명 |
|------|------|
| **입출력 제어** | 장치 드라이버 루틴들과 인터럽트 핸들러. 메모리와 디스크 시스템 사이 정보 전송 담당 |
| **기본 파일 시스템** | 장치 드라이버에게 물리 블록을 읽고 쓰도록 명령. 메모리 버퍼와 캐시를 관리 |
| **파일 구성 모듈** | 파일의 논리 블록과 물리 블록 양쪽을 알고 있음. 논리 블록 주소를 물리 블록 주소로 변환. 자유공간 관리자 포함 |
| **논리 파일 시스템** | **메타데이터** 정보를 관리. 디렉터리 구조를 관리. 파일 구조는 **FCB(File Control Block, 유닉스에서는 inode)**에 의해 관리 |

---

## 2. File-System Implementation

### A. Overview
디스크 상의 구조:
- **부트 제어 블록**: 시스템이 운영체제를 부트시키는데 필요한 정보
- **볼륨 제어 블록**: 볼륨의 블록의 수, 블록의 크기, 자유 블록의 수와 포인터 등 파티션 정보
- **디렉터리 구조**
- **파일별 FCB**

메모리 내의 정보:
- **마운트 테이블**: 마운트된 파티션 정보
- **메모리 내 디렉터리 구조**: 최근 접근된 디렉터리 정보
- **범 시스템 열린 파일 테이블**: 열린 파일들의 FCB 정보
- **프로세스별 열린 파일 테이블**: 범 시스템 열린 파일 테이블 내의 해당 항목에 대한 포인터

## 3. Directory Implementation

### A. Linear List
- 파일 이름과 데이터 블록에 대한 포인터들의 **선형 리스트**로 구현
- 가장 간단하지만 느림

### B. Hash Table
- 파일이름을 해시로 변환하여 그 값을 포인터로 활용
- 검색 속도 향상

---

## 4. Allocation Method

### A. Contiguous Allocation (연속 할당)
- 각 파일이 디스크 내에서 **연속적인 공간**을 차지
- 필요한 디스크 탐색의 횟수 최소화, 탐색 시간 최소화
- 순차 접근과 직접 접근 모두 지원

**문제점**:
- 가용공간 찾기 문제 → 동적공간 할당(최초적합, 최적 적합) → **외부 단편화** 발생
- 파일을 위한 공간을 미리 얼마나 할당할지 결정 어려움

### B. Linked Allocation (연결 할당)
- 파일은 디스크 블록의 **연결 리스트** 형태로 저장 (디스크에 흩어져 저장 가능)
- 파일 쓰기가 일어나면 자유블록을 할당받아 수행한 후 파일의 끝에 연결

**단점**:
- 순차 접근 파일에만 효과적 (직접 접근 불가 → FAT로 해결 가능)
- 포인터들을 위한 공간 필요 → 블록 단위 늘리면 내부 단편화 발생
- 신뢰성 문제: 연결 링크를 잃어버리면 데이터 손실 → 이중 연결 리스트 사용 시 오버헤드 증가

### C. Indexed Allocation (색인 할당)
- 모든 포인터들을 하나의 장소(**색인 블록**)으로 관리
- 각 파일들은 디스크 블록 주소를 모아놓은 배열인 **색인 블록**을 가짐
- 직접 접근 방식 지원

**단점**: 공간 낭비
- 대부분의 파일들은 작아서 하나 또는 두 개의 블록만으로 구성
- 각 파일들은 각자 하나의 색인 블록을 가져야 함
- **해결**: 연결 기법, 다중 수준 색인, **결합기법**

## 5. Free-Space Management

자유 공간 리스트로 자유 공간 관리:

| 방법 | 설명 |
|------|------|
| **Bit Vector** | 자유 블록이면 1, 할당되어 있으면 0 |
| **Linked List** | 자유 공간 블록들을 연결 |
| **Grouping** | 첫 번째 자유 블록 내에 n개의 블록 주소 저장. n-1개는 실제 자유 블록 주소, 마지막 하나는 다음 그룹의 주소 |
| **Counting** | 연속된 자유 블록의 첫 번째 주소와 그 뒤로 연결된 블록의 개수만 유지 |
| **Space Maps** | (고급 파일 시스템에서 사용) |
