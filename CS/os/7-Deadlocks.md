> [!tldr] TL;DR
> 교착 상태(Deadlock) - 4가지 발생 조건(상호배제·점유대기·비선점·순환대기)과 예방/회피/탐지 처리 방법.

대기중인 프로세스들이 다시는 그 상태를 변경할 수 없으면 이런 상황을 **Deadlock(교착 상태)**라고 한다.

## 1. System Model

- 시스템은 경쟁하는 프로세스들 사이에 분배되어야 할 유한한 수의 자원들로 구성 (CPU, 파일, I/O 장치들...)
- 프로세스가 자원을 사용하는 순서:
  1. **요청**: 자원을 요청. 다른 프로세스가 사용 중이면 대기
  2. **사용**: 프로세스가 자원을 사용해 요청을 수행
  3. **방출**: 프로세스가 자원을 방출

## 2. Deadlock의 특징

다음 **네 가지 조건이 동시에** 일어날 때 Deadlock이 발생할 가능성이 있다:

| 조건 | 설명 |
|------|------|
| **Mutual Exclusion** | 공유 불가능한 자원에서 상호 배제가 일어남 |
| **Hold-and-Wait** | 프로세스가 최소 하나의 자원을 점유한 채, 다른 자원을 얻기 위해 대기 |
| **No Preemption** | 자원을 사용 중인 프로세스에서 자원을 선점할 수 없음 |
| **Circular Wait** | P0, P1, ..., Pn이 서로 점유한 자원을 대기하는 순환 형태 |

## 3. 자원 할당 그래프

- 정점 V(활성 프로세스 집합 P + 자원 집합 R)와 간선 E로 구성된 그래프
- **P → R**: 프로세스가 자원 R을 사용하기 위해 대기 중
- **R → P**: 프로세스 P에 자원 R이 할당됨
- **자원할당 그래프가 Cycle로 닫힌 형태**를 보이면 교착상태

## 4. Deadlock 처리방법

### A. 교착상태 예방

교착상태 발생 조건 네 가지 중 하나라도 발생하지 않게 방지:

**상호 배제 방지**
- 여러 프로세스가 자원에 대한 동시 접근을 보장하면 상호 배제가 깨짐
- 근본적으로 공유 불가능한 자원은 예방하기 어렵다 (ex. 프린터)

**점유 대기 방지**
- 작업 수행 전에 필요한 자원들을 **모두 요청하고 획득**해야 함
- 또는 자원을 전혀 갖고 있지 않을 때만 자원 요청을 허용
- 단점: 자원의 이용률 낮아짐, 기아 상태 발생 가능

**비선점 방지**
- 서로 모든 자원에 대한 선점을 허용
- 할당받을 수 없는 자원을 요청하는 경우, 기존 자원을 모두 반납하고 재대기

**순환 대기 방지**
- 자원에 고유한 번호를 할당하고 번호 순서대로 자원을 요구하도록 함

### B. 교착 상태 회피

- 프로세스가 일생 동안 요구하고 사용할 자원에 대한 부가적인 정보를 미리 제공
- 이 정보를 바탕으로 운영체제가 어떤 요청에 대해 프로세스가 기다려야 할지 결정

**안정 상태 (Safe State)**: 교착상태를 피할 수 있음
**불안정 상태 (Unsafe State)**: 교착상태가 무조건 발생하는 것은 아니지만 발생 확률 있음

**알고리즘**:
- **자원 할당 그래프 알고리즘**: 그래프에서 교착상태 발생 여부를 판단
- **은행원 알고리즘**: 프로세스 시작 시 자신이 필요한 각 자원의 최대 개수를 선언. 요청 시 시스템이 안정 상태로 유지되는 경우에만 자원 할당. 불안정 상태가 되면 대기

### C. 교착 상태 탐지 & 회복

- 교착 상태를 허용하는 대신, 교착 상태에 빠진 것을 탐지하고 회복하는 방식

### D. 교착 상태 무시

- 유닉스와 윈도우를 포함한 대부분의 운영체제가 사용하는 방법
- 교착이 일어나는데 일어나지 않는 '척'을 한다.
