# 2. Process Management - Process

> TL;DR: 프로세스는 PCB로 추적되는 능동적 실행 단위이며, 운영체제는 상태 전이·스케쥴링·문맥 교환·IPC를 통해 다수 프로세스를 공존시킨다.

## 1. 프로세스 개념

- 메모리에 로드된 사용자 프로그램 또는 메모리 관리와 운영체제 내부의 활동들을 위해 CPU를 사용하는 활동을 모두 **프로세스**라고 부른다.
- **프로그램**: 명령어 리스트를 가진 실행 파일 (수동적 존재)
- **프로세스**: 다음에 실행할 명령어인 program counter와 관련 자원의 집합(스택, 데이터, 힙, 코드)을 가진 능동적인 존재
- 프로세스는 메모리에 **텍스트 섹션, 힙, 데이터, 스택** 영역에 적재된다.

### a. 프로세스 상태

| 상태 | 설명 |
|------|------|
| new | 프로세스가 생성 중 |
| ready | 처리기에 할당되기를 기다리는 중 |
| waiting | 어떤 사건(완료, 신호의 수신)을 기다리는 중 |
| running | 프로세스의 명령어가 실행됨 |
| terminated | 프로세스의 실행이 종료됨 |

### b. PCB (Process Control Block)

- 프로세스와 관련된 여러 정보를 가진다.
- 프로그램 카운터, CPU 레지스터들, CPU 스케쥴링 정보, 메모리 관리 정보, 입출력 상태 정보 등 프로세스마다 달라지는 상태를 저장한다.

### c. 스레드

- 한 프로세스가 다수의 스레드를 가질 수 있게 하여 한 번에 하나 이상의 일을 수행하도록 돕는다.

## 2. 프로세스 스케쥴링

- **다중 프로그래밍**의 목적: CPU의 이용을 최대화하기 위해 항상 어떤 프로세스가 실행되도록 함
- **시분할**의 목적: 각 프로그램이 실행되는 동안 사용자가 상호 작용할 수 있도록 프로세스들 사이에서 CPU를 빈번히 교체

### a. 스케쥴링 큐

- **잡 큐(Job Queue)**: 시스템 안에 있는 모든 프로세스로 구성
- **준비완료 큐(Ready Queue)**: 준비완료 상태에서 실행을 대기하는 프로세스. 연결리스트 형태로 저장
- **장치 큐(Device Queue)**: 장치 사용을 대기 중인 프로세스들의 리스트

### b. 스케쥴러

- **장기 스케쥴러**: 디스크에 저장된 프로세스들 중 어떤 것을 메모리에 적재할지 고르는 스케쥴러. 다중프로그래밍의 수 제어
- **단기 스케쥴러**: CPU를 위해 프로세스를 선택. 실행 빈도수가 높다.

### c. 문맥 교환(Context Switching)

- 인터럽트 발생 시 현재 프로세스의 문맥(CPU 레지스터 값, 프로세스 상태, 메모리 관리 정보 등)을 PCB로 저장한다.
- 문맥 교환 시간 동안엔 시스템이 유용한 일을 하지 못하므로 **순수한 오버헤드**다.

## 3. 프로세스에 대한 연산

### A. 프로세스 생성

- 생성하는 프로세스: **부모 프로세스**, 새로운 프로세스: **자식 프로세스**
- 리눅스에서 `pid`로 식별
- 자원 옵션:
  1. 모든 자원 공유
  2. 부분 공유
  3. 자원 미공유
- 실행 옵션:
  1. 자식과 부모 모두 Concurrently 실행
  2. 자식 종료 후 부모 재개

### B. 프로세스 종료

- `exit()` System call로 삭제 요청
- **좀비 프로세스**: 부모가 아직 `wait()`를 호출하지 못해 종료된 자식이 대기 중인 상태
- **고아 프로세스**: 부모가 자식의 Terminated를 wait하지 않는 경우

## 4. 프로세스 간 통신(IPC)

- **Independent Process**: 서로 데이터를 직접 주고받을 수 없다.
- **Cooperate Process**: 다른 프로세스들에게 영향을 주거나 받는 프로세스
- 협력적 프로세스 허락 이유: 정보 공유, 계산 가속화, 모듈성, 편의성

### A. 공유 메모리 시스템

- 통신하는 프로세스들이 공유 메모리 영역을 구축한다.
- 공유 메모리 사용 시 System call을 한 번만 호출한다.
- **중요 문제**: 둘이 동시에 동일한 위치에 쓰지 않도록 해야 함 → **동기화 문제**

**생산자-소비자 문제**: 공유 메모리 영역에 버퍼를 생성하여 한 쪽은 채우고 한 쪽은 소비. 임계 구역(뮤텍스, 세마포어, 락)으로 동기화 달성.

### B. 메세지 전달 시스템

- 운영체제의 메세지 전달 설비를 통한 프로세스 간 통신
- 동일한 주소 공간을 공유하지 않고도 통신하고 동기화할 수 있는 기법
